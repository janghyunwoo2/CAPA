# 멘토링 회의록 - 2026년 2월 24일

## 1. 프로젝트 개요

### 1.1 프로젝트 일정
- 최종 발표 전 발표 준비에 1주 투입

### 1.2 서비스 컨셉
- **대상**: 데이터가 필요한 세일즈 팀 (광고기획, 광고운영)
- **형태**: 웹 대시보드를 통한 모니터링 서비스
- **사용자**: 내부 사용자로 한정 (외부 사용자 제외)

### 1.3 V.O.C (Voice of Customer)
- 내부 담당자가 개발팀에 피드백 수집 및 전달

## 2. 현재 상황 및 과제

### 2.1 현재 이슈
- 내부 담당자의 SQL 지식 부재
- 메시지 큐 직접 사용의 불편함
- ERP 환경에서의 RDB 데이터 조회 어려움

### 2.2 해결 방안
- SQL 없이도 데이터 분석이 가능한 인터페이스 제공
- 데이터를 사용하기 편한 형태로 가공 및 저장

## 3. 기술 아키텍처

### 3.1 데이터 수집 및 저장
1. **Kinesis (1차 저장)**
   - 하루 동안 로그 안전 보관
   - 실시간 데이터 수집
   
2. **S3 저장소 (2차 저장)**
   - Parquet 형식으로 압축 저장
   - Firehose를 이용한 자동 전송
   - 시간 파티셔닝(dt)으로 부하 경감

### 3.2 데이터 처리
1. **ETL Summary**
   - 배치 스케줄링을 통한 자동 처리
   - 데이터를 바로 사용 가능한 형태로 가공
   - SQL 쿼리로 분석 가능한 구조 생성

2. **Athena**
   - 빅데이터 쿼리 엔진
   - 데이터 양에 따른 자동 스케일링
   - Limit 없는 대용량 데이터 조회

### 3.3 시각화
**Redash**
- 오픈소스 (무료)
- 관리가 간단함
- 많은 사용처 보유
- 사용자 친화적 인터페이스

## 4. 기술적 고려사항

### 4.1 Kinesis 설정
- **On-demand vs Provisioned**
  - On-demand: AWS가 자동으로 샤드 수 조절
  - Provisioned: 수동으로 샤드 관리 필요

### 4.2 파티션 쿼리 최적화
**올바른 사용법:**
```sql
dt >= '2026-02-16' AND dt < '2026-02-17'
-- =, >=, IN, BETWEEN AND 연산자 사용
```

**잘못된 사용법 (Full Scan 발생):**
```sql
DATE_FORMAT(dt, ...) = '2026-01'
dt LIKE '2026-01%'
```
- S3 디렉토리 전체 스캔 발생
- 스캔한 데이터 비용 + S3 List 비용 발생

### 4.3 데이터 타입
**반정형 데이터(Semi-structured Data)**
- 고정 스키마 없이 유연한 구조
- 태그나 메타데이터로 구조 정보 내재
- 예시: XML, JSON, HTML, 이메일
- NoSQL 데이터베이스에서 유연하게 저장/분석
