## 🔧 인프라 구성

### 노드 구성 전략
- **구성**: 2개의 노드로 분리 운영
- **목적**: 워크로드별 최적화 및 비용 효율성 추구

### 인스턴스 타입 분리
- **Spot 인스턴스**
  - 비용 절감을 위한 핵심 전략
  - 중단 가능한 워크로드에 활용
  - 최대 90%까지 비용 절감 가능

- **온디맨드 인스턴스**
  - 중요 서비스 전용
  - 대상 서비스:
    - Airflow 스케줄러
    - Airflow 웹서버
  - 안정성과 가용성 보장 필수

## 💰 비용 최적화 전략

### 네트워크 비용 관리
- **인바운드 트래픽**: 무료
- **아웃바운드 트래픽**: 종량제 (트래픽 기반 과금)
- **백업 시 고려사항**: 리전 간 데이터 전송 시 네트워크 비용 발생

### 로드밸런서(Load Balancer) 활용
- **LB(Load Balancer)**: 들어오는 트래픽을 여러 서버에 분산시키는 장치
  - 서버 부하 분산 및 고가용성 제공
  - 장애 발생 시 자동 페일오버
- 각 서버당 EKS 클러스터 생성
- LB로 트래픽 관리
- LB는 인바운드 처리로 비용 절감 효과
  - 인바운드 트래픽은 무료
  - 아웃바운드 비용 없음

### CDN(Content Delivery Network) 활용
- **CDN이란**: 콘텐츠 전송 네트워크로, 전 세계에 분산된 서버를 통해 사용자에게 콘텐츠를 빠르게 전달
  - 사용자와 가장 가까운 엣지 서버에서 콘텐츠 제공
  - 원본 서버의 부하 감소
  - 지연 시간(Latency) 최소화

- **비용 절감 원리**:
  - 원본 서버의 아웃바운드 트래픽 대폭 감소
  - 캐싱을 통한 반복적인 데이터 전송 최소화
  - 대역폭 비용 절약 (최대 60-80% 절감 가능)

- **클라우드플레어(Cloudflare)** 
  - 네트워크 다운로드 서비스 최적화
  - 전 세계 275개 이상의 엣지 로케이션 활용
  - DDoS 공격 방어 기능 포함
  - 무료 플랜부터 시작 가능

## 🏗️ 아키텍처 고려사항

### 고가용성(HA) 구성
- **이상적 구성**: 다중 서버 분산 처리
- **현실적 접근**: 한 서버에 집중된 구성 우선
- **점진적 확장**: 필요에 따라 확장 계획 수립

### AWS 서비스 활용
- **RDS Aurora**
  - MySQL/PostgreSQL 호환
  - 자동 확장 및 백업
  - 고가용성 기본 제공

- **Kinesis Firehose**
  - 실시간 데이터 스트리밍
  - S3, Redshift 등으로 자동 전달

- **Lambda**
  - 서버리스 컴퓨팅
  - 이벤트 기반 처리
  - 사용한 만큼만 과금
