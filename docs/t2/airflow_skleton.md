# 🛠️ LLM 도입 전, Airflow로 할 수 있는 기초 작업
LLM이 없더라도 Text-to-SQL 시스템의 뼈대를 만들기 위해 다음의 '데이터 관리 자동화' 작업들이 반드시 필요합니다.

1. DB 메타데이터 자동 추출 (스키마 수집)
- Text-to-SQL의 핵심은 "어떤 테이블에 어떤 컬럼이 있는지" 아는 것입니다.

- 작업: 매일 새벽, 운영 중인 DB에서 테이블 명세(Schema)를 뽑아서 JSON이나 CSV 파일로 저장합니다.

- 이유: 나중에 LLM을 도입했을 때, 이 파일을 읽어서 "아, 우리 회사엔 '결제' 테이블이 있구나"라고 학습하게 됩니다.

2. 데이터 품질 체크 (Validation)
- 엉망인 데이터로 SQL을 만들면 결과도 엉망이 됩니다.

- 작업: 데이터에 빈 값(Null)이 너무 많지는 않은지, 숫자가 들어가야 할 곳에 문자가 있지는 않은지 체크합니다.

- 이유: 깨끗한 데이터를 유지해야 나중에 SQL 결과값이 정확해집니다.

3. 통계 데이터 미리 계산 (Materialized View)
- 복잡한 SQL은 LLM도 만들기 힘들어합니다.

- 작업: 복잡한 조인(Join)이 필요한 데이터들을 Airflow가 밤마다 미리 합쳐서 하나의 큰 테이블(Flat Table)로 만들어 둡니다.

- 이유: LLM이 나중에 아주 쉬운 SQL만 짜도 결과를 낼 수 있게 '떠먹여 주는' 준비 과정입니다.
