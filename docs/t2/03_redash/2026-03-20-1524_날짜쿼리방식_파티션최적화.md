# 날짜 쿼리 방식 정리

## 이 문서가 설명하는 것
사용자가 보여준 쿼리의 이 부분:
```sql
WHERE 
    date_parse(concat(CAST(year AS VARCHAR), '-', 
                     LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                     LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
    BETWEEN week_start AND week_end
```
왜 이렇게 복잡하게 날짜를 조회하는지 설명합니다.

## 요청사항
- 날짜 쿼리 방식이 일간분석과 주간분석에서 다른 이유
- 시간 파티셔닝 때문에 year/month/day를 따로 지정해야 하는지 확인

## 간단한 답변
**둘 다 작동합니다. 편한 거 쓰세요!** 👍

- **방법 1**: `WHERE year='2026' AND month='03' AND day='20'` → 약간 더 빠름 (일간 분석)
- **방법 2**: `WHERE date_parse(...) BETWEEN '시작일' AND '종료일'` → 더 편함 (주간 분석)

## 왜 두 가지 방법이 있나요?

### 방법 1: 파티션 필드 직접 사용 (일간 분석에서 사용)
```sql
WHERE year = '2026' AND month = '03' AND day = '20'
```
- **장점**: 조금 더 빠름 (필요한 폴더만 읽음)
- **단점**: 여러 날짜 조회할 때 복잡함

### 방법 2: date_parse 사용 (주간 분석에서 사용)
```sql
WHERE date_parse(concat(year, '-', month, '-', day), '%Y-%m-%d') 
      BETWEEN '2026-03-14' AND '2026-03-20'
```
- **장점**: 날짜 범위 조회가 편함
- **단점**: 조금 더 느릴 수 있음 (더 많은 폴더를 읽을 수도 있음)

**👉 사용자가 보여준 쿼리가 바로 이 방식입니다!**
```sql
-- 사용자의 주간 광고주별 성과 쿼리에서 발췌
WHERE 
    date_parse(concat(CAST(year AS VARCHAR), '-', 
                     LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                     LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
    BETWEEN week_start AND week_end
```

## 그래서 뭘 써야 하나요?

### 실용적인 가이드
1. **단일 날짜 조회** → 방법 1 (year/month/day)
2. **날짜 범위 조회** → 방법 2 (date_parse)
3. **성능이 중요하지 않으면** → 편한 거 쓰세요!

### 실제 사용 예시

#### 일간 분석 (특정 날짜)
```sql
-- 2026년 3월 20일 데이터만 조회
WHERE year = '2026' AND month = '03' AND day = '20'
```

#### 주간/월간 분석 (날짜 범위)
```sql
-- 2026년 3월 14일 ~ 20일 (7일간) 데이터 조회
WHERE date_parse(concat(year, '-', month, '-', day), '%Y-%m-%d') 
      BETWEEN '2026-03-14' AND '2026-03-20'
```

## 왜 데이터가 year/month/day로 나뉘어 있나요?

S3에 데이터가 이렇게 저장되어 있습니다:
```
s3://bucket/summary/ad_combined_log_summary/
├── year=2026/
│   ├── month=03/
│   │   ├── day=01/ (이 폴더 안에 실제 데이터 파일들)
│   │   ├── day=02/ 
│   │   └── ...
```

- **파티셔닝**: 날짜별로 폴더를 나누어 저장
- **장점**: 필요한 날짜의 폴더만 읽으면 되니까 빠름
- **단점**: 날짜 조건을 잘 써줘야 함

## 결론

**네, 둘 다 작동합니다!**

사용자가 보여준 주간 광고주별 성과 쿼리는 **date_parse 방식**을 사용합니다:
- 이유: `week.start`와 `week.end` 사이의 7일치 데이터를 한 번에 조회하기 편함
- 만약 year/month/day 방식을 쓰면 7개의 조건을 일일이 써야 해서 복잡함

**현재 데이터 규모에서는 성능 차이가 거의 없으니 편한 거 쓰세요!**
