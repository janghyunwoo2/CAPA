# Redash Published/Unpublished 기능 가이드

## 개요
Redash에서 Published/Unpublished 기능은 쿼리와 대시보드의 **가시성과 접근 권한**을 제어하는 핵심 기능입니다.

## Published (게시됨) 상태

### 특징
- **다른 사용자가 볼 수 있음**: 조직 내 다른 사용자들이 쿼리를 검색하고 접근 가능
- **대시보드에 추가 가능**: Published 쿼리만 대시보드 위젯으로 추가 가능
- **공유 링크 생성 가능**: 외부 사용자와 공유할 수 있는 링크 생성
- **알림 설정 가능**: 쿼리 결과에 대한 알림 설정 가능

### 사용 시점
- 쿼리 개발이 완료되고 검증이 끝난 경우
- 팀원들과 결과를 공유해야 하는 경우
- 대시보드에 포함시켜야 하는 경우
- 정기적인 리포트로 활용하는 경우

## Unpublished (게시 취소됨) 상태

### 특징
- **작성자만 볼 수 있음**: 쿼리 작성자와 관리자만 접근 가능
- **작업 중 상태**: 아직 완성되지 않은 쿼리를 비공개로 유지
- **테스트 용도**: 쿼리를 테스트하거나 수정할 때 사용
- **대시보드 추가 불가**: Unpublished 쿼리는 대시보드에 추가할 수 없음

### 사용 시점
- 쿼리를 처음 작성하고 테스트하는 단계
- 기존 쿼리를 수정하거나 디버깅하는 경우
- 민감한 데이터를 다루는 임시 분석
- 더 이상 사용하지 않지만 삭제하기는 아까운 쿼리

## 주요 사용 시나리오

### 1. 개발 워크플로우
```
쿼리 작성 (Unpublished) 
    ↓
테스트 및 검증 (Unpublished)
    ↓
팀 리뷰 (필요시)
    ↓
공개 (Published)
```

### 2. 권한 관리
- **민감한 데이터**: 재무 정보, 개인정보가 포함된 쿼리는 신중히 관리
- **부서별 접근 제어**: 특정 부서만 볼 수 있도록 제한
- **임시 분석**: 일회성 분석은 Unpublished로 유지

### 3. 쿼리 생명주기 관리
- **활성 쿼리**: Published 상태로 유지
- **폐기 예정**: Unpublished로 변경 후 일정 기간 후 삭제
- **아카이브**: 참고용으로만 남기고 싶은 쿼리는 Unpublished

## 실제 사용 예시

### 쿼리 게시하기
1. 쿼리 편집 화면에서 "Publish" 버튼 클릭
2. 쿼리 목록에서 해당 쿼리가 공개 상태로 변경됨
3. 다른 사용자들이 검색 가능

### 쿼리 게시 취소하기
1. 쿼리 편집 화면에서 "Unpublish" 버튼 클릭
2. 대시보드에서 사용 중인 경우 경고 메시지 표시
3. 확인 후 비공개 상태로 전환

## 베스트 프랙티스

### DO
- ✅ 검증된 쿼리만 Publish
- ✅ 민감한 데이터는 신중하게 관리
- ✅ 정기적으로 사용하지 않는 쿼리 정리
- ✅ 쿼리 설명과 태그를 명확히 작성

### DON'T
- ❌ 테스트하지 않은 쿼리를 바로 Publish
- ❌ 개인정보가 포함된 쿼리를 무분별하게 공개
- ❌ 오래된 쿼리를 정리 없이 방치
- ❌ 설명 없는 쿼리를 게시

## API 활용

### 쿼리 상태 확인
```python
# Redash API를 통한 쿼리 상태 확인
GET /api/queries/{query_id}
Response: {
    "id": 123,
    "name": "Daily Revenue",
    "is_draft": false,  # false = Published
    ...
}
```

### 쿼리 게시/게시 취소
```python
# 쿼리 게시
POST /api/queries/{query_id}
Body: {"is_draft": false}

# 쿼리 게시 취소
POST /api/queries/{query_id}
Body: {"is_draft": true}
```

## 주의사항
1. **대시보드 연동**: Unpublish 시 대시보드에서 해당 쿼리 위젯이 작동하지 않음
2. **알림 중단**: Unpublished 쿼리의 알림은 자동으로 비활성화됨
3. **공유 링크 무효화**: 기존 공유 링크가 작동하지 않게 됨

## 결론
Published/Unpublished 기능은 Redash에서 협업과 데이터 거버넌스를 위한 필수 기능입니다. 적절한 활용을 통해 효율적인 쿼리 관리와 안전한 데이터 공유가 가능합니다.
