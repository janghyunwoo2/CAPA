# 슬랙 컨셉
- 슬랙에서 매일 아침(08시)마다 1개 메시지를 보낸다.
    - 1개 메시지 안에는 리포트(.pdf)와 대시보드(URL)를 제공한다.
        - pdf와 URL은 항상 한 쌍(묶음)이다.
    - 이 메시지는 주기에 따라 양이 달라진다.
        - 매달 1일 아침에는 일간과 월간이, 매주 월요일 아침에는 주간과 일간
        - 월요일이 1일이라면, 그날 아침에는 일간, 주간, 월간, 3가지 리포트가 발생한다.
    - 일간/주간/월간, 3가지 주기에 따라, 쌍이 늘어날 수 있다.
    - 모든 주기가 겹칠 경우(매달 1일), 1개 메시지 안에 3쌍이 담긴다.

# 대시보드 컨셉
- 리포트(.pdf)와 대시보드(URL)가 담겨야 한다.
    - 리포트는 일간/주간/월간 단위로 생성되며, 각 구분에 해당하는 대시보드가 URL로 보조한다.
    - 월간(매달 1일)이라면, 메시지 1건 안에 리포트+대시보드 3쌍(일간/주간/월간)이 포함된다.

# 대시보드 구성
- 일간/주간/월간 각 대시보드 1개를 만든다.
    - 각 대시보드는 `daily`, `weekly`, `monthly`를 임시 명칭으로 사용한다.
    - 이 대시보드들은 매일 오전 06시에 업데이트한다.
    - 항상 동일한 URL를 제공한다.
    - 대시보드 간 가장 큰 차이는 기간이다.
        - 기간은 최근 1일(어제 하루), 1주(지난 7일), 1개월(지난 1개월)로 구분된다.
        - 3/17 당일 기준, 일간:3/16, 주간:3/10-3/16, 월간:2/17-3/16
- 슬랙의 요청에 따라 주기가 모두 겹칠 경우, 대시보드 3건이 발생한다.
