# 주간 분석 쿼리 자동 날짜 설정 방법

## 요청사항
- 주간 분석 쿼리에서 기간을 자동으로 설정
- 종료일 파라미터 1개만 입력하면 자동으로 7일 기간 계산
- 기본값: 어제 기준 최근 7일간
- 필요시 수동으로 날짜 변경 가능

## AS-IS (현재 상태)
- 시작일과 종료일을 각각 수동으로 입력해야 함
- `{{ week_start_date }}`, `{{ week_end_date }}` 두 개의 파라미터 사용

## TO-BE (개선된 방식)
- 종료일 Date 타입 파라미터 1개만 사용 (기본값: yesterday)
- 시작일은 쿼리 내에서 자동으로 종료일-6일로 계산
- 파라미터 값을 입력하지 않아도 최근 7일 데이터 자동 조회
- 필요시 종료일만 변경하여 다른 7일 기간 분석 가능

## 구현 방법

### 1. Redash Date 파라미터의 동적 날짜 활용 (1개 파라미터)
```
Parameter: analysis_end_date
Type: Date
Default value: yesterday
Description: 분석 종료일 (기본: 어제, 시작일은 자동으로 7일전 계산)
```
**장점**:
- 파라미터 1개만 사용하여 간편함
- 시작일은 쿼리에서 자동 계산 (종료일 - 6일)
- 기본값으로 자동 실행 가능
- 필요시 종료일만 변경하면 7일 기간이 자동 조정

### 2. 실제 적용 예시

#### 자동 최근 7일 분석 쿼리 (파라미터 1개)
```sql
-- 최근 7일 광고주별 성과 - 종료일 파라미터 1개만 사용
WITH date_range AS (
    SELECT 
        DATE_ADD('day', -6, DATE '{{ analysis_end_date }}') as week_start,
        DATE '{{ analysis_end_date }}' as week_end
),
 weekly_summary AS (
    SELECT 
        advertiser_id,
        advertiser_name,
        SUM(CASE WHEN is_conversion THEN conversion_value ELSE 0 END) as weekly_revenue,
        SUM(cost_per_impression) + SUM(CASE WHEN is_click THEN cost_per_click ELSE 0 END) as weekly_cost,
        COUNT(*) as weekly_impressions,
        SUM(CASE WHEN is_click THEN 1 ELSE 0 END) as weekly_clicks,
        SUM(CASE WHEN is_conversion THEN 1 ELSE 0 END) as weekly_conversions
    FROM ad_combined_log_summary
    CROSS JOIN date_range
    WHERE 
        date_parse(concat(CAST(year AS VARCHAR), '-', 
                         LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                         LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
        BETWEEN week_start AND week_end
    GROUP BY advertiser_id, advertiser_name
)
SELECT 
    date_format(dr.week_start, '%Y-%m-%d') || ' ~ ' || 
    date_format(dr.week_end, '%Y-%m-%d') as "분석기간",
    CONCAT(SUBSTR(advertiser_id, 1, 4), '..', SUBSTR(advertiser_id, -4)) as "광고주ID",
    advertiser_name as "광고주명",
    weekly_revenue as "주간_매출",
    weekly_cost as "주간_광고비",
    weekly_impressions as "주간_노출수",
    weekly_clicks as "주간_클릭수",
    weekly_conversions as "주간_전환수",
    CASE WHEN weekly_cost > 0 
         THEN ROUND(weekly_revenue / weekly_cost, 2)
         ELSE 0 END as "주간_ROAS"
FROM weekly_summary
CROSS JOIN date_range dr
ORDER BY weekly_revenue DESC
```

#### 이번주 vs 지난주 비교 (파라미터 1개)
```sql
-- 이번주 vs 지난주 비교 - 종료일 파라미터 1개만 사용
WITH week_ranges AS (
    SELECT 
        -- 이번주 기간 (종료일 기준 7일)
        DATE_ADD('day', -6, DATE '{{ analysis_end_date }}') as this_week_start,
        DATE '{{ analysis_end_date }}' as this_week_end,
        -- 지난주 기간 (이번주보다 7일 전)
        DATE_ADD('day', -13, DATE '{{ analysis_end_date }}') as last_week_start,
        DATE_ADD('day', -7, DATE '{{ analysis_end_date }}') as last_week_end
),
weekly_data AS (
    SELECT 
        CASE 
            WHEN date_parse(concat(CAST(year AS VARCHAR), '-', 
                                 LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                                 LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
                 BETWEEN this_week_start AND this_week_end THEN '이번주(최근7일)'
            WHEN date_parse(concat(CAST(year AS VARCHAR), '-', 
                                 LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                                 LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
                 BETWEEN last_week_start AND last_week_end THEN '지난주(7일전)'
        END as period,
        SUM(CASE WHEN is_conversion THEN conversion_value ELSE 0 END) as revenue,
        SUM(cost_per_impression) + SUM(CASE WHEN is_click THEN cost_per_click ELSE 0 END) as cost,
        COUNT(*) as impressions,
        SUM(CASE WHEN is_click THEN 1 ELSE 0 END) as clicks,
        SUM(CASE WHEN is_conversion THEN 1 ELSE 0 END) as conversions
    FROM ad_combined_log_summary
    CROSS JOIN week_ranges
    WHERE 
        date_parse(concat(CAST(year AS VARCHAR), '-', 
                         LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                         LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
        BETWEEN last_week_start AND this_week_end
    GROUP BY 1
    HAVING period IS NOT NULL
)
SELECT 
    period as "기간",
    revenue as "매출",
    cost as "광고비",
    impressions as "노출수",
    clicks as "클릭수",
    conversions as "전환수",
    CASE WHEN cost > 0 THEN ROUND(revenue / cost, 2) ELSE 0 END as "ROAS",
    CASE WHEN impressions > 0 
         THEN ROUND(CAST(clicks AS DOUBLE) / impressions * 100, 2) 
         ELSE 0 END as "CTR"
FROM weekly_data
ORDER BY period DESC
```

#### 동적 7일 기간 분석 (파라미터 1개)
```sql
-- 종료일 기준 7일 기간 분석
WITH date_range AS (
    SELECT 
        DATE_ADD('day', -6, DATE '{{ analysis_end_date }}') as period_start,
        DATE '{{ analysis_end_date }}' as period_end
),
 period_summary AS (
    SELECT 
        campaign_id,
        campaign_name,
        SUM(CASE WHEN is_conversion THEN conversion_value ELSE 0 END) as revenue,
        SUM(cost_per_impression) + SUM(CASE WHEN is_click THEN cost_per_click ELSE 0 END) as cost,
        COUNT(*) as impressions,
        SUM(CASE WHEN is_click THEN 1 ELSE 0 END) as clicks
    FROM ad_combined_log_summary
    CROSS JOIN date_range
    WHERE 
        date_parse(concat(CAST(year AS VARCHAR), '-', 
                         LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                         LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
        BETWEEN period_start AND period_end
    GROUP BY campaign_id, campaign_name
)
SELECT 
    date_format(dr.period_start, '%Y-%m-%d') || ' ~ ' || 
    date_format(dr.period_end, '%Y-%m-%d') as "분석기간",
    CONCAT(SUBSTR(campaign_id, 1, 4), '..', SUBSTR(campaign_id, -4)) as "캠페인ID",
    campaign_name as "캠페인명",
    revenue as "매출",
    cost as "광고비",
    CASE WHEN cost > 0 
         THEN ROUND(revenue / cost, 2)
         ELSE 0 END as "ROAS",
    impressions as "노출수",
    clicks as "클릭수"
FROM period_summary
CROSS JOIN date_range dr
WHERE cost > 0
ORDER BY revenue DESC
LIMIT 20
```

### 3. 장점

#### 운영 효율성
- **자동화**: 매일 수동으로 날짜를 설정할 필요 없음
- **일관성**: 항상 최근 7일 데이터를 기준으로 분석
- **유연성**: 필요시 다른 기간으로 즉시 변경 가능

#### 활용 시나리오
- **일일 모니터링**: 매일 아침 최근 7일 트렌드 자동 확인
- **주간 보고서**: 금요일마다 주간 성과 자동 생성
- **특정 기간 분석**: 이벤트 기간 등 특정 7일을 선택하여 분석

### 4. 실무 활용 팁

#### 월요일 기준 주간 설정 (파라미터 1개)
```sql
-- 월요일 기준 주간 계산 (종료일 파라미터)
WITH monday_based AS (
    SELECT 
        -- 종료일이 속한 주의 월요일
        DATE_TRUNC('week', DATE '{{ analysis_end_date }}') as this_monday,
        -- 종료일이 속한 주의 일요일 (종료일)
        DATE_TRUNC('week', DATE '{{ analysis_end_date }}') + INTERVAL '6' day as this_sunday,
        -- 지난주 월요일  
        DATE_TRUNC('week', DATE '{{ analysis_end_date }}') - INTERVAL '7' day as last_monday
)
SELECT 
    this_monday as "이번주_시작",
    this_sunday as "이번주_종료",
    last_monday as "지난주_시작", 
    last_monday + INTERVAL '6' day as "지난주_종료"
FROM monday_based
```

#### 이동평균 분석 (파라미터 1개)
```sql
-- 7일 이동평균 트렌드 - 종료일 파라미터 1개 사용
WITH daily_metrics AS (
    SELECT 
        date_parse(concat(CAST(year AS VARCHAR), '-', 
                         LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                         LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d') as date,
        SUM(CASE WHEN is_conversion THEN conversion_value ELSE 0 END) as daily_revenue,
        SUM(cost_per_impression) + SUM(CASE WHEN is_click THEN cost_per_click ELSE 0 END) as daily_cost
    FROM ad_combined_log_summary
    WHERE 
        date_parse(concat(CAST(year AS VARCHAR), '-', 
                         LPAD(CAST(month AS VARCHAR), 2, '0'), '-', 
                         LPAD(CAST(day AS VARCHAR), 2, '0')), '%Y-%m-%d')
        >= DATE_ADD('day', -30, DATE '{{ analysis_end_date }}')
    GROUP BY 1
)
SELECT 
    date,
    daily_revenue,
    daily_cost,
    AVG(daily_revenue) OVER (
        ORDER BY date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as revenue_7day_avg,
    AVG(daily_cost) OVER (
        ORDER BY date 
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as cost_7day_avg
FROM daily_metrics
WHERE date >= DATE_ADD('day', -14, DATE '{{ analysis_end_date }}')
ORDER BY date DESC
```

### 5. Redash 설정 가이드

#### 기본값 설정 옵션
1. **today-N 형식**: `today-7`, `today-14`, `today-30`
2. **yesterday**: 어제 날짜
3. **this week**: 이번주 (일요일 기준)
4. **last week**: 지난주
5. **this month**: 이번달
6. **last month**: 지난달

#### 파라미터 설정 예시 (1개 파라미터만 사용)
```
# 주간 분석을 위한 Date 타입 파라미터 설정 (1개만 사용)

Name: analysis_end_date  
Type: Date
Default value: yesterday
Description: 분석 종료일 (시작일은 자동으로 종료일-6일로 계산)
```

**1개 파라미터의 장점**:
- 파라미터 1개만 관리하면 되어 간편함
- 항상 정확히 7일 기간을 보장
- 파라미터 입력 없이 기본값으로 자동 실행 (어제 기준 7일)
- 종료일만 변경하면 자동으로 7일 기간이 계산됨
- 자동 스케줄링과 수동 분석 모두 지원

**사용 예시**:
- 기본값 사용: 어제까지 최근 7일 조회 (3/13~3/19)
- 3/10 입력: 3/4~3/10까지 7일 조회
- 2/28 입력: 2/22~2/28까지 7일 조회

이렇게 설정하면 파라미터 1개만으로 간편하게 7일 주간 분석을 수행할 수 있으며, 종료일만 변경하여 다른 주간의 데이터도 쉽게 분석할 수 있습니다.
