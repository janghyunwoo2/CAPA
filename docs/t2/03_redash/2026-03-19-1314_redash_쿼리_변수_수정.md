## TO-BE (변경 후)

### 0. 집계 함수 변경 (가장 중요)
| 기존 쿼리 | 실제 쿼리로 변경 | 설명 |
|----------|-----------------|------|
| `SUM(total_impressions)` | `COUNT(*)` | 각 행이 하나의 impression |
| `SUM(total_clicks)` | `SUM(CASE WHEN is_click THEN 1 ELSE 0 END)` | is_click 플래그로 집계 |
| `SUM(total_cost)` | `SUM(cost_per_impression) + SUM(CASE WHEN is_click THEN cost_per_click ELSE 0 END)` | 실제 비용 계산 |
| `SUM(total_revenue)` | `SUM(CASE WHEN is_conversion THEN conversion_value ELSE 0 END)` | 전환 수익 (일별 테이블) |
| `SUM(total_revenue)` | `SUM(CASE WHEN is_click THEN 15000.0 ELSE 0 END)` | 클릭당 예상 수익 (시간별 테이블) |
| `SUM(total_conversions)` | `SUM(CASE WHEN is_conversion THEN 1 ELSE 0 END)` | is_conversion 플래그로 집계 |

### 1. 비용(Cost) 계산 방식
```sql
-- 기존: SUM(total_cost)
-- 변경: SUM(total_impressions * 5.0 + total_clicks * 300.0)
```
- CPM (Cost Per Mille): 5,000원 → impression당 5원으로 계산
- CPC (Cost Per Click): 300원
- 실제 광고 시장의 평균 단가 반영

### 2. 수익(Revenue) 계산 방식
```sql
-- 일간/시간별 분석: SUM(total_clicks * 15000.0)  -- 클릭당 평균 매출 15,000원
-- 월간 분석: SUM(total_conversions * 30000.0)  -- 전환당 평균 매출 30,000원
```

## 추가 수정 (2026-03-19 13:27)

### 날짜 파라미터 LPAD/CAST 적용
- **AS-IS**: 날짜 파라미터를 숫자 타입으로 직접 사용
- **TO-BE**: LPAD와 CAST를 사용하여 문자열 변환 후 사용

#### 수정 이유
- S3 파티션은 `year='2026'`, `month='03'`, `day='19'` 형식으로 저장됨
- Redash 숫자 파라미터는 03을 3으로 변환하여 파티션 조회 실패
- 텍스트 파라미터는 보안상 대시보드 공유 불가

#### 수정 패턴
```sql
-- AS-IS
WHERE 
    year = {{ year|2026 }}
    AND month = {{ month|3 }}
    AND day = {{ day|19 }}

-- TO-BE  
WHERE 
    year = CAST({{ year|2026 }} AS VARCHAR)
    AND month = LPAD(CAST({{ month|3 }} AS VARCHAR), 2, '0')
    AND day = LPAD(CAST({{ day|19 }} AS VARCHAR), 2, '0')
```

#### 적용 내역
1. **일간분석.md**: 모든 WHERE 절의 날짜 파라미터에 LPAD/CAST 적용
2. **월간분석.md**: 모든 WHERE 절의 날짜 파라미터에 LPAD/CAST 적용 (day 제외, 월간이므로)
3. **주간분석.md**: 이미 date_parse 내부에서 LPAD/CAST 사용 중이므로 WHERE 절만 수정

#### 주의사항
- date_parse 함수 내부에서는 이미 LPAD/CAST가 적용되어 있어 중복 적용 불필요
- 파라미터 타입은 여전히 Number로 유지 (보안상 안전)
- Athena/Presto 환경에서 검증 완료

#### 수정 패턴
```sql
-- AS-IS (오류 발생)
COUNT(*) as 총_노출수

-- TO-BE (정상 작동)
COUNT(*) as "총_노출수"
```

#### 수정 범위
1. **일간분석.md**: 7개 쿼리의 모든 한글 alias에 큰따옴표 추가
   - WITH절 내부의 컬럼명에도 큰따옴표가 있는 경우, SELECT절에서 참조시에도 큰따옴표 필요

#### 주의사항
- **큰따옴표(") 사용**: 작은따옴표(')가 아닌 큰따옴표 사용

#### 주요 스키마 오류 수정 내역

1. **월간분석.md - 광고비 효율성 분석 (쿼리 4.4)**

   -- AS-IS (오류: 중복 WHERE 절)
   WITH advertiser_summary AS (

       WHERE year = ... AND month = ...
   )

   ```

   WHERE total_impressions > 10000
   
   -- TO-BE (수정됨)
   COUNT(*) as "노출수"
   HAVING COUNT(*) > 1000  -- WHERE → HAVING 변경
   ```

3. **일간/주간/월간 공통 수정사항**

#### 확인된 작동 쿼리
- ✅ 일간 KPI 요약: 한글 alias + 큰따옴표 + 집계 함수 정상 작동
- ✅ 시간별 트렌드: COUNT(*) 및 SUM(CASE WHEN...) 정상 작동
- ✅ 월간 목표 달성률: conversion_value 활용 정상 작동
- ✅ 주간 비교: 파티션 조건 및 날짜 계산 정상 작동
