# 광고 분석 기초지식 가이드

## 1. 기초지식 및 배경설명

### 1.1 광고 데이터 구조
우리가 분석하는 광고 데이터는 다음과 같은 계층 구조를 가집니다:

```
광고주(Advertiser) 
  └─> 캠페인(Campaign)
       └─> 광고그룹(Ad Group)
            └─> 광고(Ad)
```

예시:
- 광고주: "삼성전자"
- 캠페인: "갤럭시 S24 출시 캠페인"
- 광고그룹: "20대 타겟", "30대 타겟"
- 광고: "유튜브 동영상 광고 A", "인스타그램 이미지 광고 B"

### 1.2 핵심 지표 의미

#### 📊 **노출수 (Impressions)**
- **의미**: 광고가 사용자에게 보여진 횟수
- **예시**: 유튜브에서 광고가 1,000번 재생됨 = 노출수 1,000

#### 🖱️ **클릭수 (Clicks)**
- **의미**: 사용자가 광고를 클릭한 횟수
- **예시**: 1,000번 노출 중 50번 클릭 = 클릭수 50

#### 💰 **광고비 (Cost)**
- **의미**: 광고를 집행하는데 사용한 비용
- **예시**: 하루에 100만원 광고비 사용

#### 💵 **매출 (Revenue)**
- **의미**: 광고를 통해 발생한 수익
- **예시**: 광고를 보고 제품을 구매해서 300만원 매출 발생

#### 🛒 **전환 (Conversions)**
- **의미**: 광고를 보고 실제 구매나 가입을 한 횟수
- **예시**: 50번 클릭 중 5명이 구매 = 전환수 5

### 1.3 계산 지표 이해하기

#### 📈 **CTR (Click Through Rate, 클릭률)**
```
CTR = (클릭수 ÷ 노출수) × 100
```
- **의미**: 광고를 본 사람 중 몇 %가 클릭했는지
- **예시**: 노출 1,000번, 클릭 50번 → CTR = 5%
- **해석**: CTR이 높으면 광고가 매력적이라는 의미

#### 💸 **CPC (Cost Per Click, 클릭당 비용)**
```
CPC = 광고비 ÷ 클릭수
```
- **의미**: 클릭 1번당 드는 비용
- **예시**: 광고비 10만원, 클릭 100번 → CPC = 1,000원
- **해석**: CPC가 낮을수록 효율적

#### 🎯 **CVR (Conversion Rate, 전환율)**
```
CVR = (전환수 ÷ 클릭수) × 100
```
- **의미**: 클릭한 사람 중 몇 %가 구매했는지
- **예시**: 클릭 100번, 구매 5번 → CVR = 5%
- **해석**: CVR이 높으면 광고 타겟팅이 정확하다는 의미

#### 📊 **ROAS (Return On Ad Spend, 광고 수익률)**
```
ROAS = 매출 ÷ 광고비
```
- **의미**: 광고비 1원당 발생한 매출
- **예시**: 광고비 100만원, 매출 300만원 → ROAS = 3.0
- **해석**: ROAS가 1보다 크면 이익, 1보다 작으면 손해

### 1.4 핵심 SQL 함수 설명

#### 📅 **날짜 관련 함수**
```sql
-- 텍스트를 날짜로 변환
date_parse('2026-03-19', '%Y-%m-%d')

-- 날짜를 원하는 형식의 텍스트로 변환
date_format(날짜컬럼, '%Y-%m-%d')

-- 여러 텍스트를 하나로 합치기
concat('2026', '-', '03', '-', '19')  -- 결과: '2026-03-19'

-- 왼쪽에 특정 문자로 채우기
LPAD('3', 2, '0')  -- 결과: '03'
```

#### 🧮 **집계 함수**
```sql
SUM(total_impressions)     -- 모든 값을 더하기
COUNT(DISTINCT advertiser_id)  -- 중복 제거하고 개수 세기
AVG(total_cost)           -- 평균 구하기
```

#### 🔍 **조건 함수**
```sql
-- CASE WHEN: 조건별로 다른 결과 반환
CASE 
    WHEN SUM(total_impressions) > 0 
    THEN ROUND(CAST(SUM(total_clicks) AS DOUBLE) / SUM(total_impressions) * 100, 2)
    ELSE 0 
END as ctr

-- CAST: 데이터 타입 변환 (정수→실수)
CAST(total_clicks AS DOUBLE)

-- ROUND: 반올림
ROUND(3.14159, 2)  -- 결과: 3.14
```

### 1.5 데이터베이스 구조
- **데이터베이스**: `ad_analytics`
- **주요 테이블**:
  - `ad_combined_log`: 시간별 데이터
  - `ad_combined_log_summary`: 일별 집계 데이터
- **파티션**: year, month, day, hour로 구분

### 1.6 Redash 파라미터 템플릿
```sql
-- 공통 파라미터 템플릿
{{ year|2026 }}              -- 연도

{{ start_date|2026-03-01 }}  -- 시작일
{{ end_date|2026-03-19 }}    -- 종료일
{{ advertiser_id }}          -- 광고주 ID (선택사항)
{{ campaign_id }}            -- 캠페인 ID (선택사항)
{{ min_impressions|1000 }}   -- 최소 노출수 필터
```

## 5. 대시보드 구성 및 운영 가이드

### 5.1 대시보드 레이아웃

#### 일간 대시보드
```
[Row 1: KPI Cards - 4개 나란히]
- 총 노출수 | 총 클릭수 | 총 광고비 | 총 매출

[Row 2: 성과 지표 - 4개 나란히]  
- CTR(%) | CPC(원) | CVR(%) | ROAS

[Row 3: 차트 영역 - 2개 분할]
- 시간별 트렌드 (좌측 70%)
- CPC 분포 (우측 30%)

[Row 4: 테이블]
- Top 20 광고주 성과 테이블 (전체 너비)
```

#### 주간 대시보드
```
[Row 1: 주간 성과 카드]
- 주간 총 매출 | 주간 ROAS | 전주 대비 성장률 | 활성 광고주 수

[Row 2: 비교 차트]
- 이번주 vs 지난주 (Bar Chart) | 요일별 패턴 (Area Chart)

[Row 3: 상세 분석]
- 광고주별 주간 성과 피벗 테이블

[Row 4: 효율성 지표]
- CTR vs CVR 산점도 | CPC 트렌드
```

#### 월간 대시보드
```
[Row 1: 목표 달성 Progress Bar]
- 매출 달성률 | 전환 달성률 | 예산 소진율 | 월간 ROAS

[Row 2: 일별 히트맵]
- 일별 성과 히트맵 (매출 강도)

[Row 3: 트렌드 차트]
- 일별 매출 with 7일 이동평균

[Row 4: Top 성과]
- Top 20 광고주 테이블
```

### 5.2 권한 설정
- **일간 대시보드**: 전체 공개
- **주간 대시보드**: 팀장급 이상
- **월간 대시보드**: 경영진

### 5.3 자동 새로고침 설정
- 일간: 1시간마다
- 주간: 6시간마다  
- 월간: 24시간마다

### 5.4 성능 최적화
1. **파티션 조건 필수 사용**
   ```sql
   WHERE year = 2026 AND month = 3 AND day = 19
   ```

2. **필요한 컬럼만 선택**
   ```sql
   SELECT advertiser_id, SUM(revenue) -- 필요한 것만
   ```

3. **적절한 캐싱 활용**
   - 자주 변하지 않는 데이터는 캐시 시간 늘리기

### 5.5 트러블슈팅

#### 쿼리 타임아웃
```sql
-- 해결: 날짜 범위 축소 또는 샘플링
WHERE ... 
AND MOD(ABS(HASH(ad_id)), 100) < 10  -- 10% 샘플링
```

#### 0으로 나누기 오류 방지
```sql
CASE WHEN denominator > 0 
     THEN numerator / denominator
     ELSE 0 END
```

#### NULL 값 처리
```sql
COALESCE(SUM(revenue), 0)  -- NULL을 0으로 변환
```

## 6. 주의사항 및 팁

### 6.1 자주 하는 실수
1. **정수 나누기**: CAST AS DOUBLE 사용
2. **0으로 나누기**: CASE WHEN으로 체크
3. **NULL 처리**: COALESCE 사용
4. **파티션 누락**: 성능 저하 주의

### 6.2 최적화 팁
1. 큰 날짜 범위는 피하기
2. DISTINCT는 필요한 경우만
3. 서브쿼리보다 WITH 절 활용
4. 인덱스가 있는 컬럼으로 필터링

### 6.3 데이터 검증
- ETL 완료 시간 확인
- 타임존 설정 확인 (KST)
- 파티션 업데이트 상태 확인
