# Airflow DAG용 독립적인 ETL 모듈 구성

## 요구사항
- 기존 etl_summary_t2 폴더를 수정하지 않고 독립적인 버전 생성
- Airflow DAG에서 import 오류 없이 사용 가능한 구조

## AS-IS (현재 상태)
### 현재 구조
```
services/data_pipeline_t2/
├── dags/                    # Airflow DAG 파일
│   └── 01_ad_hourly_summary.py
├── etl_summary_t2/          # 원본 ETL 모듈 (수정 금지)
│   ├── __init__.py
│   ├── hourly_etl.py
│   ├── daily_etl.py
│   ├── athena_utils.py
│   └── config.py
└── ...
```

### 현재 문제
- Airflow 컨테이너에서 etl_summary_t2 모듈을 찾을 수 없음
- 경로 계산 오류로 인한 import 실패

## TO-BE (독립 모듈 구성)
### 새로운 구조
```
services/data_pipeline_t2/
├── dags/                    
│   ├── 01_ad_hourly_summary.py
│   ├── 02_ad_daily_summary.py
│   └── etl_modules/         # 독립적인 ETL 모듈 (새로 생성)
│       ├── __init__.py
│       ├── hourly_etl.py
│       ├── daily_etl.py
│       ├── athena_utils.py
│       └── config.py
├── etl_summary_t2/          # 원본 (유지)
└── ...
```

## 구현 방안

### 1. dags/etl_modules 폴더 생성
- DAG 파일과 같은 위치에 모듈 배치
- Airflow가 자동으로 인식 가능한 경로

### 2. 필요한 파일만 복사
- 원본 etl_summary_t2에서 필요한 파일만 선택적 복사
- 상대 import를 절대 import로 변경

### 3. DAG 파일 수정
```python
# 01_ad_hourly_summary.py
from etl_modules.hourly_etl import HourlyETL
from etl_modules.config import *
```

## 장점
- 원본 코드 보존
- Airflow 환경에 최적화된 구조
- Docker 볼륨 마운트 불필요
- 즉시 적용 가능
