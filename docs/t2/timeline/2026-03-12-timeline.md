#### ✅ 해결 방법
etl_summary_t2 패키지를 직접 사용하도록 변경:
```python
# sys.path에 패키지 경로 추가
ETL_PACKAGE_PATH = os.path.join(os.path.dirname(os.path.dirname(__file__)), 'etl_summary_t2')
if ETL_PACKAGE_PATH not in sys.path:
    sys.path.insert(0, os.path.dirname(ETL_PACKAGE_PATH))

# HourlyETL 직접 실행
def _run_hourly_etl(**context):
    from etl_summary_t2.hourly_etl import HourlyETL
    dt_utc = context.get('data_interval_end')
    dt_kst = pendulum.instance(dt_utc).in_timezone('Asia/Seoul')
    etl = HourlyETL(target_hour=dt_kst)
    etl.run()
```

### 5️⃣ 15:29 - 시간대 처리 검증

#### 🔍 확인 사항
UTC(Airflow) ↔ KST(ETL) 시간대 변환 정확성

#### ✅ 검증 결과
```python
# 시간대 변환 프로세스
# UTC: 2026-03-12T06:00:00+00:00
# KST: 2026-03-12T15:00:00+09:00
```

3. ETL 패키지: KST로 실행
4. S3 파티션: KST 기준으로 생성

### 6️⃣ 15:38 - Airflow API 버전 문제

#### 🔴 문제 발견
```python
TypeError: Invalid arguments were passed to PythonOperator (task_id: create_hourly_summary). 
Invalid arguments were: **kwargs: {'provide_context': True}
```

#### 🔍 원인 분석
- Airflow 3.1.7에서 `provide_context` 파라미터 deprecated
- Airflow 2.0 이상부터 자동으로 context 전달

#### ✅ 해결 방법
모든 DAG 파일에서 `provide_context=True` 제거:
```python
# 수정 전
create_hourly_summary = PythonOperator(
    task_id="create_hourly_summary",
    python_callable=_run_hourly_etl,
    provide_context=True,  # 제거 필요
)

# 수정 후
create_hourly_summary = PythonOperator(
    task_id="create_hourly_summary",
    python_callable=_run_hourly_etl,
)
```

### 9️⃣ 16:44 - Schedule Interval API 변경

#### 🔴 문제 발견
```
TypeError: DAG.__init__() got an unexpected keyword argument 'schedule_interval'
```

#### 🔍 원인 분석
Airflow 2.4 이상에서 `schedule_interval`이 `schedule`로 변경

#### ✅ 해결 방법
```python
# 수정 전
with DAG(
    dag_id='01_ad_hourly_summary',
    schedule_interval='0 * * * *',
    ...
)

# 수정 후
with DAG(
    dag_id='01_ad_hourly_summary',
    schedule='0 * * * *',
    ...
)
```
