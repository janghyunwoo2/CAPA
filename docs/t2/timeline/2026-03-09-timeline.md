### 1. Raw 데이터 및 파티셔닝 문제 (오전 초반)

#### 문제 발견
- **증상**: 
  - S3에 raw 데이터가 없음 (`s3://capa-data-lake-827913617635/raw/impressions/year=2026/month=03/day=09/` 비어있음)
  - Athena에서 데이터 조회 시 0건 반환
  - Summary 데이터가 파티셔닝 없이 단순 폴더로 저장됨

#### 원인 분석
1. **Raw 데이터 부재**
   - 로그 생성기(`gen_adlog_t2/`)가 실행되지 않음
   - 로컬 환경에서 명시적으로 실행 필요

2. **파티션 구조 문제**
   - 현재: `s3://bucket/summary/ad_combined_log/dt=2026-03-09-06/` (단순 폴더)
   - 필요: `s3://bucket/summary/ad_combined_log/year=2026/month=03/day=09/hour=06/` (파티션 구조)

#### 해결 방법
- Raw 데이터 생성 스크립트 실행
- 파티션 구조 변경 계획 수립

#### 해결 여부
- ⏳ 부분 해결 (원인 파악 완료, 실행 대기)

### 2. Glue 카탈로그 중복 테이블 문제 (오전 중반)

#### 문제 발견
- **증상**: 
  - 파티션별로 개별 테이블이 생성됨 (`ad_combined_log_2026_03_01`, `ad_combined_log_2026_03_02` 등)
  - Athena에서 통합 테이블 조회 불가
  - Glue 카탈로그 오염

#### 원인 분석
1. **테이블 스키마 불일치**
   - 기존 테이블: `dt` (단일 파티션)
   - 코드 기대: `year/month/day/hour` (4개 파티션)

2. **쿼리 실패로 인한 연쇄 반응**
   - DESCRIBE 실패 → CTAS 시도 → TABLE_ALREADY_EXISTS 에러
   - 파티션별 새 테이블 생성

#### 해결 방법
- Glue 카탈로그 정리
- 테이블 스키마 통일

#### 해결 여부
- ✅ 해결 (코드 수정으로 해결)

### 3. Glue-Athena 테이블 연동 문제 (오전 후반)

#### 문제 발견
- **증상**: 
  - S3에는 데이터가 있지만 Athena에서 조회 안 됨
  - 임시 테이블(`ad_combined_log_temp_*`)이 계속 생성됨
  - 쿼리 타임아웃 발생

#### 원인 분석
1. **파티션 미등록**
   - CTAS로 S3에 직접 쓰기 → Glue 카탈로그 미동기화
   - `MSCK REPAIR TABLE` 누락

2. **임시 테이블 축적**
   - 매시간 새로운 임시 테이블 생성
   - DROP TABLE 실패 또는 누락

#### 해결 방법
- 파티션 자동 등록 추가 (`MSCK REPAIR TABLE`)
- 임시 테이블 자동 정리 로직 추가

#### 해결 여부
- ✅ 해결 (파티션 수리 로직 추가)

### 4. ETL 파티션 스키마 수정 (오후 초반)

#### 문제 발견
- **증상**: 
  - DELETE 쿼리: `COLUMN_NOT_FOUND` (year 컬럼 없음)
  - INSERT 쿼리: `TYPE_MISMATCH` (컬럼 수 불일치)

#### 원인 분석
- **스키마 불일치 확정**
  - 실제 테이블: `dt` 파티션
  - 코드: `year/month/day/hour` 파티션 사용

#### 해결 방법
```python
# 변경 전
SELECT ..., '{self.year}' AS year, '{self.month}' AS month, ...
WHERE year = '{self.year}' AND month = '{self.month}' ...

# 변경 후
SELECT ..., '{self.hour_str}' AS dt   # hour_str = "2026-03-09-03"
WHERE dt = '{self.hour_str}'
```
