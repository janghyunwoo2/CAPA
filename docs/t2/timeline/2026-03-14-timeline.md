# 2026-03-14 일과 타임라인 정리

**작성일**: 2026-03-14  
**주요 이슈**: Athena CSV 파일 생성 문제 해결

---

## 📋 전체 요약
S3 summary 폴더에 CSV/metadata 파일이 지속적으로 쌓이는 문제를 분석하고 해결방안을 도출한 날. Terraform Workgroup 설정이 Python 코드 설정을 무시하는 근본 원인을 발견하고 즉시 적용 가능한 해결책을 구현함.

---

## 🔍 문제 인식

### 1. CSV/Metadata 파일 누적 문제
- **현상**: S3 summary 폴더에 CSV와 metadata 파일이 지속적으로 쌓임
- **영향**: 
  - 일일 약 50개 파일 생성 (월 1,500개)
  - S3 저장 공간 낭비 및 비용 증가
  - 폴더 구조 복잡화로 성능 저하 우려
- **발생 시점**: ETL 프로세스 실행 시마다 발생

### 2. 경로 설정 불일치
- **의도**: Python 코드에서 `.athena-temp/` 경로 사용
- **실제**: CSV 파일이 `athena-results/` 경로에 생성됨
- **영향**: 임시 파일 격리 실패

---

## 🔬 원인 분석

### 1. Terraform Workgroup 강제 설정 (근본 원인)
```hcl
# terraform/08-athena.tf
resource "aws_athena_workgroup" "capa" {
  configuration {
    enforce_workgroup_configuration = true  # ❗ 핵심: 강제 설정
    result_configuration {
      output_location = "s3://${aws_s3_bucket.data_lake.bucket}/athena-results/"
    }
  }
}
```

### 2. Python 코드 설정 무시
```python
# Python에서 설정한 경로가 무시됨
ResultConfiguration={'OutputLocation': 's3://bucket/.athena-temp/'}
```

### 3. WorkGroup 미지정 문제
- Python 코드에서 WorkGroup 파라미터 누락
- 기본 워크그룹(`capa-workgroup`) 자동 적용
- 강제 설정에 의해 Python 설정 덮어씌워짐

### 4. 파일 생성 흐름
1. **ETL 실행** → Athena 쿼리 실행
2. **워크그룹 강제 설정** → `athena-results/` 경로 사용
3. **CSV 파일 생성** → 매 쿼리마다 {query_id}.csv 생성
4. **누적** → 자동 삭제 없이 계속 쌓임

## ✅ 해결 방법

### 1. 즉시 해결: Primary Workgroup 명시적 지정
```python
# 모든 start_query_execution()에 WorkGroup='primary' 추가
response = self.client.start_query_execution(
    QueryString=query,
    QueryExecutionContext={'Database': database},
    ResultConfiguration={'OutputLocation': ATHENA_TEMP_RESULTS_PATH},
    WorkGroup='primary'  # ✅ 추가: Terraform 설정 우회
)
```

**수정 대상 파일**:
- `services/data_pipeline_t2/etl_summary_t2/athena_utils.py`
- `services/data_pipeline_t2/dags/etl_modules/athena_utils.py`
- `services/report-generator/src/athena_utils.py`

### 2. 장기 해결 방안

#### 방안 A: 워크그룹 설정 변경
```hcl
# Terraform 수정 옵션 1: 강제 설정 해제
enforce_workgroup_configuration = false

# Terraform 수정 옵션 2: 출력 경로 변경
output_location = "s3://${aws_s3_bucket.data_lake.bucket}/.athena-temp/"
```

#### 방안 B: 다중 워크그룹 전략
- ETL 전용 워크그룹 생성
- 용도별 워크그룹 분리 (ETL, 콘솔, 분석)

#### 방안 C: S3 라이프사이클 정책
```json
{
  "Rules": [{
    "Id": "DeleteAthenaResults",
    "Status": "Enabled",
    "Filter": {"Prefix": "athena-results/"},
    "Expiration": {"Days": 7}
  }]
}
```

### 3. 추가 개선사항
- UNLOAD 명령으로 전환 검토 (CSV 중간 파일 없이 직접 Parquet 저장)
- 쿼리 실행 후 즉시 파일 삭제 로직 추가

## 📊 성과 및 영향

### 개선 효과
- **즉시**: CSV 파일이 `.athena-temp/` 경로에 생성
- **비용**: S3 PUT 요청 비용 절감
- **성능**: 폴더 구조 단순화로 리스트 작업 성능 향상
- **관리**: 자동화된 파일 정리 가능

### 학습 사항
1. AWS Athena Workgroup의 `enforce_workgroup_configuration` 설정은 매우 강력함
2. WorkGroup을 명시하지 않으면 기본 워크그룹이 자동 적용됨
3. Primary workgroup은 일반적으로 강제 설정이 비활성화되어 있음
