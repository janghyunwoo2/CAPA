## 1. Daily ETL date/datetime 타입 오류 (09:13)

### 문제 상황
- DailyETL 클래스가 datetime 객체를 받아 `.date()` 메서드를 호출하려 했으나 실패
- 오류: `'datetime.date' object has no attribute 'date'`

### 원인 분석
- DAG에서 이미 `.date()` 객체를 전달
- DailyETL 생성자에서 또다시 `.date()` 호출 시도

### 해결 방법
```python
# DailyETL.__init__ 메서드 수정
if hasattr(target_date, 'date'):
    self.target_date = target_date.date()
elif isinstance(target_date, date):
    self.target_date = target_date
else:
    self.target_date = target_date
```

  - 처리할 데이터: 11시

## 9. Schedule과 Data Interval 문제 해결 (13:04)

### 문제 상황
- 스케줄 변경 후 현재 시간 데이터를 처리하려고 시도

### 해결 방법
```python
# 01_ad_hourly_summary.py 수정
dt_utc = context["data_interval_start"]
dt_kst = pendulum.instance(dt_utc).in_timezone('Asia/Seoul')

# 실제 처리할 시간: data_interval_start - 1시간
target_hour_kst = dt_kst.subtract(hours=1)

etl = HourlyETL(target_hour=target_hour_kst)
```
