## 🏗️ Text-to-SQL에서 Airflow가 필요한 3가지 이유

### 1. AI의 학습 재료(데이터)를 신선하게 유지하기

Text-to-SQL 모델이 정확한 쿼리를 짜려면 우리 회사의 최신 데이터베이스 구조(스키마)를 잘 알고 있어야 합니다.

- **문제:** 데이터베이스 구조는 수시로 변합니다.
- **Airflow의 역할:** 매일 밤 자동으로 바뀐 DB 구조를 추출해서 AI 모델이 참고할 수 있도록 업데이트(색인 생성) 작업을 수행합니다.

### 2. 복잡한 전처리 과정의 자동화 (파이프라인)

자연어를 SQL로 바꾸기 전후에는 여러 단계의 작업이 필요합니다.

- **과정:** `사용자 질문 검사` → `SQL 생성` → `SQL 문법 검사` → `실제 DB 실행` → `결과를 다시 자연어로 요약`.
- **Airflow의 역할:** 이 여러 단계 중 하나라도 실패하면(예: DB 연결 오류) 다시 실행하거나, 관리자에게 알람을 보내 프로세스가 꼬이지 않게 관리합니다.

### 3. 모델 성능 모니터링 및 재학습

AI가 만든 SQL이 정답이었는지 매번 기록하고 평가해야 성능이 좋아집니다.

- **Airflow의 역할:** 사용자의 질문과 AI의 답변을 모아서 성능 지표를 계산하고, 일정 수치 이하로 떨어지면 "모델 재학습"이라는 무거운 작업을 자동으로 예약해서 실행합니다.
