# Ad Log Generator 데이터 생성량 비교 분석

## 개요
백필용 `ad_log_generator.py`와 실시간용 `main.py`의 데이터 생성량에 현저한 차이가 발생하는 원인을 분석합니다.

## 1. 데이터 생성량 비교

### 1.1 ad_log_generator.py (백필용)
- **기본 시간당 노출 수**: 10,000개
- **트래픽 멀티플라이어**: 시간대/요일별 0.1 ~ 3.0배
- **실제 시간당 노출 수**: 1,000 ~ 30,000개 (평균 약 15,000개)
- **1일 총 노출 수**: 약 360,000개

### 1.2 main.py (실시간용)
- **생성 주기**: 1초에 1개
- **시간당 노출 수**: 3,600개 (고정)
- **1일 총 노출 수**: 86,400개

### 1.3 생성량 차이
- **백필용이 실시간용보다 약 4.2배 많음**
- 백필: 1일 36만 건 vs 실시간: 1일 8.6만 건

## 2. 상세 차이점 분석

### 2.1 생성 속도
| 구분 | 백필용 (ad_log_generator.py) | 실시간용 (main.py) |
|------|------------------------------|-------------------|
| 생성 방식 | 배치 생성 (시간당 한번에) | 스트림 생성 (1초마다) |
| 시간당 기본 | 10,000개 | 3,600개 |
| 피크 시간대 | 최대 30,000개 (점심/저녁) | 3,600개 (고정) |
| 새벽 시간대 | 최소 1,000개 | 3,600개 (고정) |

### 2.2 트래픽 패턴
#### 백필용 트래픽 멀티플라이어
```python
# 시간대별 패턴 (ad_log_generator.py)
- 00~07시: 0.1 ~ 0.2배 (1,000 ~ 2,000개)
- 07~09시: 0.4 ~ 0.6배 (4,000 ~ 6,000개)  
- 11~14시: 1.5 ~ 2.0배 (15,000 ~ 20,000개) # 점심 피크
- 17~21시: 2.0 ~ 3.0배 (20,000 ~ 30,000개) # 저녁 피크

# 요일별 추가 가중치
- 평일(월~목): 0.8 ~ 1.0배
- 금요일: 1.2 ~ 1.5배
- 주말: 1.3 ~ 2.0배
```

#### 실시간용 트래픽
- **고정**: 시간/요일 관계없이 1초에 1개

### 2.3 CTR/CVR 비교
| 메트릭 | 백필용 | 실시간용 |
|--------|--------|----------|
| CTR | 1~5% (ad_format별 차등) | 10% (고정) |
| CVR | 1~10% (conversion_type별 차등) | 20% (고정) |

## 3. 문제점 및 영향

### 3.1 데이터 불균형
- 백필 데이터가 실시간보다 4배 이상 많아 통계 왜곡 가능
- 실시간 데이터가 피크 시간대 트래픽을 반영하지 못함

### 3.2 비현실적인 패턴
- 실시간: 새벽 시간대에도 동일한 트래픽 (비현실적)
- 실시간: CTR 10%, CVR 20%는 매우 높은 수치

### 3.3 테스트/분석 영향
- 시간대별 분석 시 실시간 데이터는 패턴이 없음
- 전체 데이터 분석 시 백필 데이터에 편향됨

## 4. 개선 방안

### 4.1 단기 개선안 (실시간 생성 속도 조정)
```python
# main.py 수정 예시
def get_generation_interval():
    """시간대별 생성 간격 반환 (초)"""
    hour = datetime.now().hour
    
    if 0 <= hour < 7:
        return random.uniform(3.0, 4.0)  # 새벽: 3~4초마다
    elif 11 <= hour < 14:
        return random.uniform(0.2, 0.3)  # 점심: 0.2~0.3초마다
    elif 17 <= hour < 21:
        return random.uniform(0.1, 0.2)  # 저녁: 0.1~0.2초마다
    else:
        return random.uniform(0.8, 1.2)  # 평시: 0.8~1.2초마다

# 메인 루프에서
time.sleep(get_generation_interval())
```

### 4.2 중기 개선안 (설정 통합)
- 두 생성기가 동일한 설정(시간대별 패턴, CTR/CVR)을 사용하도록 통합
- 환경 변수나 설정 파일로 관리

### 4.3 장기 개선안 (생성기 통합)
- 백필과 실시간을 하나의 생성기로 통합
- 모드 선택: `--mode batch` (백필) / `--mode stream` (실시간)
- 공통 트래픽 패턴 및 비즈니스 로직 공유

## 5. 즉시 조치 사항

### 5.1 실시간 생성량 증가
main.py에서 생성 간격을 줄여 시간당 생성량 증가:
```python
# 현재: time.sleep(1.0) → 시간당 3,600개
# 변경: time.sleep(0.5) → 시간당 7,200개
# 또는: time.sleep(0.3) → 시간당 12,000개
```

### 5.2 트래픽 패턴 추가
백필과 유사한 시간대별 트래픽 패턴을 실시간에도 적용하여 현실성 향상

## 6. 결론
현재 실시간 데이터 생성기는 백필 대비 약 1/4 수준의 데이터만 생성하고 있으며, 시간대별 트래픽 패턴도 없어 비현실적입니다. 단기적으로는 생성 속도를 높이고, 중장기적으로는 두 생성기의 로직을 통합하는 것이 필요합니다.
