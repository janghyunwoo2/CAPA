# 03_ad_hourly_summary_test & 04_ad_daily_summary_test 작동방식 매뉴얼

## 개요
본 문서는 Airflow DAG 파일인 `03_ad_hourly_summary_test.py`와 `04_ad_daily_summary_test.py`의 작동 방식을 설명합니다.
이 DAG들은 테스트 및 수동 실행을 위해 설계되었으며, 정식 버전인 01*, 02* DAG의 간소화된 버전입니다.

## 파일 구조 변경 사항

### AS-IS (변경 전)
- 복잡한 KubernetesPodOperator 지원
- ETL 로직이 스크립트 문자열로 정의
- 여러 개의 Task와 복잡한 의존성
- 조건부 실행 로직 (USE_KPO)
- ExternalTaskSensor 사용

### TO-BE (변경 후)
- PythonOperator만 사용
- 독립 ETL 모듈 직접 import
- 단일 Task 구조로 단순화
- 불필요한 의존성 제거
- 명확한 실행 흐름

## 03_ad_hourly_summary_test.py 작동 방식

### 역할
- **목적**: HourlyETL을 수동으로 실행하여 시간별 광고 요약 데이터 생성
- **주요 기능**: impressions + clicks 테이블을 조인하여 ad_combined_log 테이블 생성
- **실행 방식**: 수동 트리거 전용 (schedule=None)

### 실행 프로세스
1. **ETL 모듈 import**
   ```python
   from etl_modules.hourly_etl import HourlyETL
   ```

2. **Task 실행 함수**
   ```python
   def run_hourly_etl(**context):
       # 수동 테스트 전용: 현재 시간에서 1시간 전으로 동적 계산
       current_kst = pendulum.now('Asia/Seoul')
       dt_kst = current_kst.subtract(hours=1).replace(minute=0, second=0, microsecond=0)
       print(f"[DEBUG] Current KST time: {current_kst}")
       print(f"[DEBUG] ETL Target KST time: {dt_kst} (현재 시간 - 1시간)")
       etl = HourlyETL(target_hour=dt_kst)
       etl.run()
   ```

3. **DAG 구성**
   - 단일 PythonOperator Task
   - 수동 실행 전용 설정
   - 에러 처리 및 로깅 포함

## 04_ad_daily_summary_test.py 작동 방식

### 역할
- **목적**: DailyETL을 수동으로 실행하여 일별 광고 요약 데이터 생성
- **주요 기능**: 24시간치 ad_combined_log + conversions 테이블 집계
- **실행 방식**: 수동 트리거 전용 (schedule=None)

### 실행 프로세스
1. **ETL 모듈 import**
   ```python
   from etl_modules.daily_etl import DailyETL
   ```

2. **Task 실행 함수**
   ```python
   def run_daily_etl(**context):
       # 수동 실행 시 전날 날짜 사용
       dt = context.get("data_interval_start", pendulum.now('Asia/Seoul').subtract(days=1))
       # DailyETL은 date 객체를 받음
       target_date = dt.date() if hasattr(dt, 'date') else dt
       etl = DailyETL(target_date=target_date)
       etl.run()
   ```

3. **DAG 구성**
   - 단일 PythonOperator Task
   - 수동 실행 전용 설정
   - 날짜 변환 로직 포함

## 주요 차이점: 정식 버전(01*, 02*) vs 테스트 버전(03*, 04*)

| 구분 | 정식 버전 (01, 02) | 테스트 버전 (03, 04) |
|------|-------------------|---------------------|
| 스케줄 | 정기 실행 (매시간/매일) | 수동 실행 전용 |
| 구조 | 최소 구성 | 최소 구성 |
| ETL Import | etl_modules | etl_modules |
| Task 개수 | 1개 | 1개 |
| 용도 | 프로덕션 | 테스트/디버깅 |

## 실행 방법

### Airflow UI에서 수동 실행
1. Airflow 웹 UI 접속
2. DAGs 목록에서 `03_ad_hourly_summary_test` 또는 `04_ad_daily_summary_test` 찾기
3. "Trigger DAG" 버튼 클릭
4. 실행 파라미터 확인 후 실행

### CLI에서 실행
```bash
# Hourly 테스트 실행
airflow dags trigger 03_ad_hourly_summary_test

# Daily 테스트 실행
airflow dags trigger 04_ad_daily_summary_test
```

## 트러블슈팅

### 문제: ModuleNotFoundError
- **원인**: etl_modules 경로를 찾을 수 없음
- **해결**: DAG 파일이 올바른 위치에 있는지 확인

### 문제: 시간대 불일치
- **원인**: UTC/KST 변환 문제
- **해결**: pendulum 라이브러리가 자동으로 처리하며, DAG는 Asia/Seoul 시간대 사용

### 문제: ETL 실행 실패
- **원인**: AWS 인증, S3 권한, Athena 쿼리 오류 등
- **해결**: 로그를 확인하여 상세 오류 메시지 파악

## 모니터링
1. **Task 로그 확인**: Airflow UI에서 Task 클릭 → View Log
2. **ETL 내부 로그**: [INFO], [SUCCESS], [ERROR] 태그로 구분
3. **실행 시간**: execution_timeout으로 제한 (hourly: 30분, daily: 60분)

## 주의사항
- 테스트 DAG는 수동 실행 전용이므로 프로덕션에서는 01*, 02* DAG 사용
- ETL 모듈이 S3 파티션 관리까지 처리하므로 별도의 MSCK REPAIR TABLE 불필요
- 실행 전 AWS 자격증명과 권한이 올바르게 설정되어 있는지 확인
