# Python Import 절대경로 vs 상대경로 문제 해결 (2026-03-12)

## 문제 상황
```bash
PS C:\Users\Dell5371\Desktop\projects\CAPA\services\data_pipeline_t2> python -m etl_summary_t2.run_etl daily --target-date 2026-03-03
ModuleNotFoundError: No module named 'athena_utils'
```

### AS-IS (문제 발생)
- Python 모듈 실행 시 `athena_utils` 모듈을 찾을 수 없는 오류 발생
- `etl_summary_t2` 패키지 내부에서 절대 import 사용
  ```python
  # hourly_etl.py
  from athena_utils import AthenaQueryExecutor
  from config import DATABASE, S3_PATHS, ...
  ```
- 이전에는 작동했지만, 세션이나 환경이 변경되면서 오류 발생

### TO-BE (문제 해결)
- 패키지 내부 모듈 간에는 상대 import 사용
  ```python
  # hourly_etl.py
  from .athena_utils import AthenaQueryExecutor
  from .config import DATABASE, S3_PATHS, ...
  ```

## 근본 원인
1. **Python 모듈 검색 경로 문제**
   - `python -m etl_summary_t2.run_etl` 실행 시 Python은 `etl_summary_t2`를 패키지로 인식
   - 패키지 내부에서 절대 import 사용 시 최상위 경로에서 모듈을 찾으려 시도
   - `athena_utils`가 최상위 경로에 없어서 오류 발생

2. **이전에 작동한 이유 (추정)**
   - VS Code Python 확장이 작업 공간 디렉토리를 자동으로 Python path에 추가
   - 또는 PYTHONPATH 환경 변수가 설정되어 있었음
   - 재부팅이나 새 세션에서 이 설정이 초기화됨

## 해결 방법

### 변경된 파일들
1. **hourly_etl.py**
   - `from athena_utils import ...` → `from .athena_utils import ...`
   - `from config import ...` → `from .config import ...`

2. **daily_etl.py**
   - `from athena_utils import ...` → `from .athena_utils import ...`
   - `from config import ...` → `from .config import ...`

3. **athena_utils.py**
   - `from config import ...` → `from .config import ...`

4. **run_etl.py**
   - `from hourly_etl import ...` → `from .hourly_etl import ...`
   - `from daily_etl import ...` → `from .daily_etl import ...`

## 교훈
- **패키지 내부 모듈 간**: 항상 상대 import (`.module_name`) 사용
- **패키지 외부 모듈**: 절대 import 사용
- **명확한 구조**: 패키지로 설계된 경우 상대 import로 의존성을 명시
- **환경 독립성**: PYTHONPATH나 특정 환경 설정에 의존하지 않도록 구현
