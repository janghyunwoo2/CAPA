## 🔍 [1단계] 데이터 파이프라인 정합성 검증

### 1) 테스트 환경 및 조건
- **데이터 소스**: 로컬 `Log Generator` (Python)
- **수집 인프라**: AWS Kinesis Data Stream + Firehose
- **저장소**: S3 Data Lake (Parquet Format)
- **메타데이터**: AWS Glue Catalog (Crawler 방식)
- **쿼리 엔진**: AWS Athena

### 2) 검증 상세 결과
- **Log Generation**: 약 30분간 실행하여 실시간 로그 생성 및 전송.
- **S3 적재 확인**: `s3://capa-data-lake-xxx/raw/` 경로에 26개의 Parquet 파일 생성 확인.
- **자동 스키마 인식**: `capa-log-crawler`를 통해 `ad_events_raw` 테이블 및 `year/month/day` 파티션 자동 생성 성공.
- **데이터 정합성 (Athena 쿼리)**:
    - **총 레코드 수**: **1,504건** (충분한 테스트 표본 확보)
    - **스키마 매핑**: `event_id`, `event_type`, `timestamp`, `bid_price` 등 모든 컬럼이 정상 타입으로 매핑됨.
    - **조회 샘플**:
        ```json
        {
          "event_id": "3c3a7257-568c-4794-b3dc-f729e5ec4b5c",
          "event_type": "impression",
          "device_type": "iOS",
          "bid_price": 318.15
        }
        ```

### 3) 판정: ✅ **Pass**
> 로그 생성부터 Athena 쿼리까지 끊김 없는 데이터 흐름을 확인하였으며, 특히 Crawler 도입을 통해 스키마 관리 자동화까지 검증 완료함.
