## 2. E2E 시나리오

```
[Data Flow]
Log Generator → Kinesis Stream → Firehose → S3 (Parquet)
     ↓
  Glue Catalog → Athena

[App Flow]
Slack Bot Echo → (향후) Athena 쿼리 → Slack 응답
Airflow → (향후) DAG 실행 → Report 생성
```

---

## 3. 검증 단계

### 3.1 데이터 파이프라인 검증

#### Step 1: Log Generator 실행 중 확인

```powershell
# 로컬에서 실행 중인 generator.py 확인
# 또는
ps | Select-String "python"
```

#### Step 2: Kinesis Metrics 확인

```powershell
# CloudWatch Metrics
aws cloudwatch get-metric-statistics `
    --namespace AWS/Kinesis `
    --metric-name IncomingRecords `
    --dimensions Name=StreamName,Value=capa-stream `
    --start-time (Get-Date).AddMinutes(-10).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ss") `
    --end-time (Get-Date).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ss") `
    --period 300 `
    --statistics Sum

# Sum > 0 확인
```

#### Step 3: S3 Parquet 파일 확인

```powershell
# S3 최신 파일 확인
aws s3 ls s3://capa-data-lake-<ACCOUNT_ID>/raw/ --recursive --human-readable | Select-Object -Last 10
```

#### Step 4: Athena 쿼리 실행

```powershell
# Athena 쿼리
$Query = "SELECT event_type, COUNT(*) as count FROM ad_events_raw GROUP BY event_type"

aws athena start-query-execution `
    --query-string $Query `
    --query-execution-context Database=capa_db `
    --result-configuration OutputLocation=s3://capa-data-lake-<ACCOUNT_ID>/athena-results/
```

---

### 3.2 EKS Apps 검증

#### Step 5: EKS Pods 상태 확인

```powershell
# 모든 Pod 확인
kubectl get pods --all-namespaces

# 예상 출력:
# NAMESPACE   NAME                            READY   STATUS
# airflow     airflow-webserver-xxxxx         1/1     Running
# airflow     airflow-scheduler-xxxxx         2/2     Running
# default     slack-bot-xxxxx                 1/1     Running
```

#### Step 6: Airflow Webserver 접속

```powershell
# URL 확인
kubectl get svc -n airflow airflow-webserver

# 브라우저에서 접속: http://<EXTERNAL-IP>:8080
# 로그인: admin / admin
```

#### Step 7: Slack Bot Echo 테스트

**Slack 채널에서**:
```
@capa-bot echo E2E Test Success!

# 예상 응답:
# Echo: E2E Test Success!
```

### 3.3 전체 흐름 검증 (최종)

#### Step 8: 데이터 생성 → Athena 조회 → Slack 응답 (수동 시뮬레이션)

1. **Log Generator 실행** (1분간)
   ```powershell
   python services\log-generator\generator.py
   # Ctrl+C로 1분 후 중지
   ```

2. **Athena에서 최신 데이터 확인**
   ```sql
   SELECT * FROM ad_events_raw
   WHERE year = '2026' AND month = '02' AND day = '11'
   ORDER BY timestamp DESC
   LIMIT 10;
   ```

3. **Slack Bot에서 Echo 테스트**
   ```
   @capa-bot echo Pipeline Working!
   ```

4. **Airflow DAG List 확인**
   - Webserver UI에서 DAG 목록 확인
