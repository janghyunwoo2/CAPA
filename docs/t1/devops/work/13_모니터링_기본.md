## 3. 실행 단계

### 3.1 CloudWatch Logs 확인

#### Kinesis Firehose Logs

```powershell
# Firehose 로그 그룹 확인
aws logs describe-log-groups --query "logGroups[?contains(logGroupName, 'firehose')].logGroupName"

# 최근 로그 확인
aws logs tail /aws/kinesisfirehose/capa-firehose --since 10m
```

#### EKS Control Plane Logs

```powershell
# EKS API Server 로그
aws logs tail /aws/eks/capa-eks/cluster --since 10m
```

### 3.2 CloudWatch Metrics Dashboard 생성 (선택)

**AWS Console**:
1. CloudWatch → Dashboards → Create dashboard
2. 위젯 추가:
   - **Kinesis Stream Metrics**
     - Namespace: AWS/Kinesis
     - Metric: IncomingRecords, IncomingBytes
     - Dimension: StreamName = capa-stream
   
   - **EKS Node Metrics**
     - Namespace: ContainerInsights
     - Metric: node_cpu_utilization, node_memory_utilization
     - Dimension: ClusterName = capa-eks

### 3.3 EKS Pod Logs 확인

```powershell
# Airflow Scheduler 로그
kubectl logs -n airflow deployment/airflow-scheduler --tail=50

# Slack Bot 로그
kubectl logs -l app=slack-bot --tail=50

# 모든 Namespace Pod 상태
kubectl get pods --all-namespaces | Select-String -NotMatch "Running|Completed"
# 예상: 출력 없음 (모두 Running)
```

### 3.4 Kinesis Stream 상태 확인

```powershell
# Stream 상태
aws kinesis describe-stream-summary --stream-name capa-stream

# 예상 출력:
# StreamStatus: "ACTIVE"
# OpenShardCount: 1
```

### 3.5 S3 사용량 확인

```powershell
# S3 Bucket 용량 (대략적)
aws s3 ls s3://capa-data-lake-<ACCOUNT_ID>/raw/ --recursive --summarize

# 예상 출력:
# Total Objects: 123
# Total Size: 5.2 MB
```

## 4. CloudWatch Alarm 설정 (기본)

### 4.1 Kinesis Stream Record 감소 Alarm

```powershell
# Alarm 생성
aws cloudwatch put-metric-alarm `
    --alarm-name capa-kinesis-low-records `
    --alarm-description "Kinesis 레코드 수가 감소하면 알림" `
    --metric-name IncomingRecords `
    --namespace AWS/Kinesis `
    --statistic Sum `
    --period 300 `
    --threshold 10 `
    --comparison-operator LessThanThreshold `
    --evaluation-periods 1 `
    --dimensions Name=StreamName,Value=capa-stream

# 확인
aws cloudwatch describe-alarms --alarm-names capa-kinesis-low-records
```

### 4.2 EKS Node CPU 높음 Alarm (선택)

```powershell
aws cloudwatch put-metric-alarm `
    --alarm-name capa-eks-high-cpu `
    --metric-name node_cpu_utilization `
    --namespace ContainerInsights `
    --statistic Average `
    --period 300 `
    --threshold 80 `
    --comparison-operator GreaterThanThreshold `
    --evaluation-periods 2 `
    --dimensions Name=ClusterName,Value=capa-eks
```

## 5. 검증 방법

### 5.1 CloudWatch Metrics 조회

```powershell
# Kinesis Metrics (최근 1시간)
aws cloudwatch get-metric-statistics `
    --namespace AWS/Kinesis `
    --metric-name IncomingRecords `
    --dimensions Name=StreamName,Value=capa-stream `
    --start-time (Get-Date).AddHours(-1).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ss") `
    --end-time (Get-Date).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ss") `
    --period 300 `
    --statistics Sum
```

### 5.2 EKS Pod 상태 확인

```powershell
# 모든 Pod 확인
kubectl get pods --all-namespaces -o wide

# Pods 개수 확인
kubectl get pods --all-namespaces --no-headers | Measure-Object | Select-Object Count
```
