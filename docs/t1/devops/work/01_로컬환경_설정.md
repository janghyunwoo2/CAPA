# 작업 01: 로컬 개발 환경 설정

> **Phase**: 0 (사전 준비)  
> **담당**: DevOps Engineer  
> **예상 소요**: 15분  
> **선행 작업**: 없음

---

## 1. 목표

로컬 환경에서 Terraform, AWS CLI, kubectl 등 필수 도구가 설치되고 작동하는지 확인합니다.

---

## 2. 필수 도구 및 버전

| 도구 | 최소 버전 | 설치 확인 명령어 | 용도 |
|------|----------|-----------------|------|
| **AWS CLI** | 2.x | `aws --version` | AWS 리소스 관리 |
| **Terraform** | 1.5+ | `terraform version` | IaC 코드 실행 |
| **kubectl** | 1.28+ | `kubectl version --client` | EKS 클러스터 관리 |
| **Helm** | 3.x | `helm version` | Kubernetes 앱 배포 |
| **Docker** | 20.x+ | `docker --version` | 컨테이너 이미지 빌드 |

---

## 3. 실행 단계

### 3.1 도구 설치 확인

```powershell
# 1. AWS CLI
aws --version
# 예상 출력: aws-cli/2.x.x Python/3.x.x Windows/...

# 2. Terraform
terraform version
# 예상 출력: Terraform v1.5.x or later

# 3. kubectl
kubectl version --client
# 예상 출력: Client Version: v1.28.x or later

# 4. Helm
helm version
# 예상 출력: version.BuildInfo{Version:"v3.x.x"}

# 5. Docker
docker --version
# 예상 출력: Docker version 20.x.x or later
```

### 3.2 AWS 자격 증명 설정

```powershell
# AWS 자격 증명 확인
aws sts get-caller-identity

# 예상 출력:
# {
#     "UserId": "AIDAXXXXXXXXXXXXXXXXX",
#     "Account": "123456789012",
#     "Arn": "arn:aws:iam::123456789012:user/your-user"
# }
```

**AWS 자격 증명이 없는 경우**:
```powershell
aws configure
# AWS Access Key ID [None]: YOUR_ACCESS_KEY
# AWS Secret Access Key [None]: YOUR_SECRET_KEY
# Default region name [None]: ap-northeast-2
# Default output format [None]: json
```

> ⚠️ **보안 주의**: Production 환경에서는 OIDC 사용 (Access Key 금지). 로컬 개발에서만 임시로 사용.

### 3.3 기본  리전 설정 확인

```powershell
# 기본 리전 확인
aws configure get region
# 예상 출력: ap-northeast-2
```

---

## 4. 검증 방법

### 4.1 모든 도구 버전 출력

```powershell
Write-Host "=== 환경 검증 ===" -ForegroundColor Green
Write-Host "AWS CLI: $(aws --version)" -ForegroundColor Cyan
Write-Host "Terraform: $(terraform version)" -ForegroundColor Cyan
Write-Host "kubectl: $(kubectl version --client --short)" -ForegroundColor Cyan
Write-Host "Helm: $(helm version --short)" -ForegroundColor Cyan
Write-Host "Docker: $(docker --version)" -ForegroundColor Cyan

# AWS 자격 증명
Write-Host "`n=== AWS 자격 증명 ===" -ForegroundColor Green
aws sts get-caller-identity
```

### 4.2 성공 기준

- [ ] 모든 명령어가 오류 없이 실행됨
- [ ] `aws sts get-caller-identity`가 유효한 계정 정보 반환
- [ ] 리전이 `ap-northeast-2`로 설정됨

---

## 5. 실패 시 대응

| 오류 | 원인 | 해결 방법 |
|------|------|-----------|
| `aws: command not found` | AWS CLI 미설치 | [AWS CLI 설치 가이드](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html) |
| `terraform: command not found` | Terraform 미설치 | [Terraform 설치](https://developer.hashicorp.com/terraform/downloads) |
| `Unable to locate credentials` | AWS 자격 증명 미설정 | `aws configure` 실행 |
| `kubectl: command not found` | kubectl 미설치 | [kubectl 설치](https://kubernetes.io/docs/tasks/tools/) |
