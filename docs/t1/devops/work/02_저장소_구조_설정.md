## 3. 실행 단계

### 3.1 프로젝트 루트로 이동

```powershell
cd "c:\Users\Jang_home\Desktop\git tool\CAPA"
```

### 3.2 폴더 구조 생성

```powershell
# Infrastructure 폴더 생성
New-Item -ItemType Directory -Force -Path "infrastructure\terraform\environments\dev\base"
New-Item -ItemType Directory -Force -Path "infrastructure\terraform\environments\dev\apps"
New-Item -ItemType Directory -Force -Path "infrastructure\terraform\modules"
New-Item -ItemType Directory -Force -Path "infrastructure\helm-values"

# Services 폴더 생성
New-Item -ItemType Directory -Force -Path "services\slack-bot"
New-Item -ItemType Directory -Force -Path "services\log-generator"

# 확인
Get-ChildItem -Recurse -Directory infrastructure
```

### 3.3 .gitignore 파일 생성

**루트에 `.gitignore` 생성** (이미 있다면 추가):

```powershell
# .gitignore에 추가할 내용
@"
# Terraform
*.tfstate
*.tfstate.backup
.terraform/
.terraform.lock.hcl
terraform.tfvars
*.tfvars.json

# AWS
.aws/
credentials

# 환경 변수
.env
.env.local
*.local

# Python
__pycache__/
*.py[cod]
*.so
.Python
venv/
.venv/

# IDEs
.vscode/
.idea/
*.swp
*.swo

# OS
.DS_Store
Thumbs.db

# Logs
*.log

# Secrets
secrets.yaml
*-secrets.yaml
"@ | Out-File -FilePath ".gitignore" -Encoding UTF8 -Append

Write-Host ".gitignore 파일 업데이트 완료" -ForegroundColor Green
```

### 3.4 README 파일 생성

```powershell
# infrastructure/README.md
@"
# CAPA Infrastructure

## 구조

- **terraform/environments/dev/base/**: AWS 리소스 (Kinesis, S3, EKS, IAM)
- **terraform/environments/dev/apps/**: Kubernetes 앱 (Airflow, Slack Bot)
- **helm-values/**: Helm Chart Values

## 배포 순서

1. base: \`cd infrastructure/terraform/environments/dev/base && terraform apply\`
2. apps: \`cd infrastructure/terraform/environments/dev/apps && terraform apply\`

## 참고 문서

- [DevOps Implementation Guide](../../docs/t1/devops/devops_implementation_guide.md)
"@ | Out-File -FilePath "infrastructure\README.md" -Encoding UTF8

Write-Host "README.md 생성 완료" -ForegroundColor Green
```

## 4. 검증 방법

### 4.1 폴더 구조 확인

```powershell
# 트리 구조 출력
tree infrastructure /F
```

**예상 출력**:
```
infrastructure
├── helm-values
├── terraform
│   ├── environments
│   │   └── dev
│   │       ├── apps
│   │       └── base
│   └── modules
└── README.md
```

### 4.2 .gitignore 동작 확인

```powershell
# 테스트 파일 생성
"test" | Out-File -FilePath "infrastructure\terraform\environments\dev\base\terraform.tfstate"

# Git 상태 확인 (*.tfstate가 무시되어야 함)
git status

# 테스트 파일 삭제
Remove-Item "infrastructure\terraform\environments\dev\base\terraform.tfstate"
```

### 4.3 성공 기준

- [ ] `infrastructure/terraform/environments/dev/base/` 폴더 존재
- [ ] `infrastructure/terraform/environments/dev/apps/` 폴더 존재
- [ ] `infrastructure/helm-values/` 폴더 존재
- [ ] `.gitignore`에 `*.tfstate`, `.env` 등 포함
- [ ] `git status`에서 `.tfstate` 파일이 무시됨

## 5. 실패 시 대응

| 오류 | 원인 | 해결 방법 |
|------|------|-----------|
| `New-Item: Path already exists` | 폴더가 이미 존재 | `-Force` 옵션 사용 (덮어쓰기) |
| `.gitignore` 무시 안 됨 | Git 캐시 문제 | `git rm --cached <file>` 실행 |
