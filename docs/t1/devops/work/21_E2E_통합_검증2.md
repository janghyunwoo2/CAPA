## 🔍 [2단계] Airflow 오케스트레이션 및 서비스 연동 검증

### 1) 테스트 환경 및 조건
- **워크플로우 엔진**: Airflow (Karpenter 프로비저닝 노드 배치)
- **실행 방식**: `KubernetesPodOperator` (KPO)를 통한 태스크 실행
- **검증 목표**: 스팟 노드 및 코어 노드 위에서 파드가 정상적으로 구동되고 EKS 내부 통신이 가능한지 확인

### 2) 검증 항목
- [x] **웹서버 접근 확인**: Airflow Web UI 로드밸런서 접속 정상 여부
- [x] **DAG 파싱 상태**: GitSync를 통해 최신 DAG 정상 로드 확인
- [x] **의존성(KPO) 태스크 실행**:
    - [x] `athena_aggregation_pod` 태스크 스케줄링 및 Athena 쿼리 성공 여부
    - [x] `trigger_report_pod` 태스크를 통한 Report Generator 서비스(내부 DNS) 통신 성공 여부
- [x] **권한(IRSA) 유지 확인**: 파드 재생성 후에도 S3/Athena 접근 권한이 정상 유지되는지 검증

- [x] **ChromaDB 데이터 영속성**: Vector DB 데이터 유실 확인 및 재학습(DDL, Doc, SQL) 완료
- [x] **Anthropic 모델 호출**: API 키 정상 연동 및 SQL 변환 성공 확인 (Vanna API 정상 가동 중)

### 3.2) Report Generator 서비스 연동

- [x] **체인 연동 (AI)**: Report Generator -> Vanna AI 내부 API 통신 정상 여부 (200 OK 확인)
