
만약 두번째 질문에 첫번째 질문과 관련없는거 넣으면
 어떻게 동작하나?

정상적으로 동작합니다.

  1번째: "어제 기기별 클릭수"
  2번째: "이번달 캠페인별 ROAS 알려줘"
    → Step 0: history=1건 (이전 이력 있음)
    → Step 2 정제: "이번달 캠페인별 ROAS(광고
  수익률)"  ← 이전 맥락 무시하고 독립 질문 처리
    → Redash: 5건 반환 ✅

  이전 이력이 있어도 관련 없는 질문은 정상적으로 독립
   처리됩니다. QuestionRefiner가 이전 대화와
  연결고리가 없으면 그냥 현재 질문만 정제하고, 이전
  session_id, turn_number만 올라갑니다.

  즉 현재 동작:

  ┌──────────────────┬───────────────────────────┐
  │       경우       │   QuestionRefiner 동작    │
  ├──────────────────┼───────────────────────────┤
  │ "그 중에서",     │ 이전 맥락 참조 → 구체적   │
  │ "두번째" 등      │ 조회 조건으로 변환        │
  │ 지시어           │                           │
  ├──────────────────┼───────────────────────────┤
  │ 완전히 새로운    │ 이전 맥락 무시 → 독립     │
  │ 질문             │ 질문으로 정제             │
  └──────────────────┴──────────────────────────

왜 바차트만 나오나

  AI Analyzer 프롬프트 기준:
  "bar":  categorical comparisons (campaign/platform
  ranking)  ← 현재 질문들이 전부 여기 해당
  "line": time series (daily/weekly/monthly trend)

  지금까지 한 질문들:

- "어제 기기별 클릭수" → 기기 비교 → bar
- "캠페인별 CTR" → 캠페인 비교 → bar
- "지역별 노출수 순위" → 지역 비교 → bar

  모두 카테고리 비교 형태라 bar가 맞습니다.

### [멀티턴 구현 및 동작 원리 확인]

1. **Step 0 (ConversationHistoryRetriever)**:
   - 후속 질문 시 가장 먼저 실행되는 단계.
   - `MULTI_TURN_ENABLED=true`일 때, 사용자의 `session_id`를 기반으로 **DynamoDB**(`capa-dev-query-history`)에서 이전 대화 이력을 조회하여 맥락(`conversation_history`)을 구성함.

2. **Step 11 (History Recorder)**:
   - 파이프라인의 최종 단계.
   - `DYNAMODB_ENABLED=true`일 때, 현재 질문, 생성된 SQL, AI의 답변(`answer`), `session_id`, `turn_number` 등을 **DynamoDB**에 저장함.
   - **핵심 역할**: 이 단계에서 데이터를 저장해야만 다음번 질문의 **Step 0**에서 맥락을 이어받을 수 있음.

> **결론**: Step 11에서 DynamoDB에 저장하는 행위가 선행되어야 다음 질문의 Step 0에서 멀티턴 맥락을 인식할 수 있게 됩니다. (실제 코드 `main.py` 및 `query_pipeline.py` 로직 확인 완료)
