## Elevator Pitch (30초)

> **"AI 분석 플랫폼은 분석 경험을 개선하고, 데이터 파이프라인은 신뢰성을 보장합니다."**

CAPA는 광고 비즈니스 비개발자가 **슬랙에서 자연어로 질문하면 AI가 SQL을 생성**하여 즉시 답변하는 플랫폼입니다. 데이터 조회 시간을 **3시간~3일 → 1분**으로 단축하고, 이상 징후를 실시간으로 감지합니다.

### 3. Solution Introduction (해결 방안) - 1분

#### CAPA: AI 기반 분석 플랫폼

**4가지 핵심 기능**:

| 기능 | 설명 | 임팩트 |
|------|------|--------|
| 🗣️ **Ask** | 슬랙에서 자연어 질문 → AI가 SQL 생성 | 3시간 → 1분 |
| 📝 **Report** | 주간/월간 리포트 자동 생성 | 4시간 → 1분 |
| 🔔 **Alert** | 이상 탐지 + 즉시 알림 | D+1 → 실시간 |
| 📊 **Dashboard** | KPI 대시보드 (Redash) | SQL 불필요 |

**IAB 보고서 지적 → CAPA 해결**:

| 지적 | CAPA 해결 |
|------|-----------|
| 속도 부족 | Text-to-SQL로 1분 내 조회 |
| 정확성 부족 | AI가 SQL 생성 (휴먼 에러 제거) |
| 신뢰성 부족 | 실시간 이상 탐지 |

### 5. Key Features Demo (핵심 기능 시연) - 2분

#### Ask 기능

**데모 시나리오**:
```
사용자: "@capa-bot 어제 캠페인별 전환율 top 10"

CAPA 응답:
📊 결과:
| campaign_id | cvr   |
|-------------|-------|
| camp_A      | 5.2%  |
| camp_B      | 4.8%  |
...

🔍 생성된 SQL:
SELECT campaign_id, 
       ROUND(COUNT(conversion_id) * 100.0 / 
             COUNT(click_id), 1) as cvr
FROM events
WHERE date = '2026-02-10'
GROUP BY 1
ORDER BY cvr DESC
LIMIT 10

📌 기준:
- CVR = 전환수 / 클릭수
- 기간: 2026-02-10 (어제)
```

**핵심 포인트**:
- ✅ 투명성: 생성된 SQL 공개 → 결과 검증 가능
- ✅ 즉시성: 1분 내 응답
- ✅ 정확성: RAG 기반 도메인 용어 이해

#### Report 기능

**자동 생성 리포트 예시**:
```markdown
# 📊 주간 광고 성과 리포트

## 📈 주요 KPI
- 노출수: 10M (+5% WoW)
- 클릭수: 250K (+3% WoW)
- 전환수: 12,500 (-2% WoW)

## 🏆 Top 캠페인
1. camp_A (CTR 2.8%)
2. camp_B (CTR 2.5%)

## 💡 AI 인사이트
전환율이 소폭 감소했습니다. 
랜딩페이지 최적화를 권장합니다.
```

#### Alert 기능

**이상 탐지 흐름**:
```
Prophet 시계열 예측
  → 기대값: CTR 2.3% (± 0.3%)
  → 실제값: CTR 1.2%
  → 신뢰구간 벗어남
  → 🚨 Slack 알림
```

### 6. 기술 스택 & 차별화 - 1분

#### 기술 스택

```
[Data Pipeline]
Kinesis → Firehose → S3 (Parquet) → Glue → Athena

[AI/ML]
- Text-to-SQL: Vanna AI + ChromaDB + GPT-4
- Alert: CloudWatch (MVP) / Prophet (확장)

[Infrastructure]
Terraform + EKS + Airflow

[Interface]
Slack Bolt + FastAPI + Redash
```

#### 기술적 차별화

| 특징 | 설명 |
|------|------|
| **RAG 기반 Text-to-SQL** | ChromaDB에 DDL, 예시 SQL 학습 → 정확도 향상 |
| **투명성** | AI가 생성한 SQL을 사용자에게 공개 → 검증 가능 |
| **MVP/확장 분리** | 리스크 최소화 (LLM 없이 파이프라인 먼저 검증) |
| **시계열 예측 Alert** | 단순 임계값이 아닌 Prophet으로 계절성 반영 |

**왜 Vanna AI인가?**
- 오픈소스 Text-to-SQL 프레임워크
- ChromaDB + LLM 조합으로 RAG 구현 간소화
- 학습 데이터 관리 용이

## Q&A 예상 질문

### Q1. Vanna AI의 정확도는?
**A**: 벤치마크 기준 80-90%. RAG로 도메인 예시 SQL을 학습시켜 정확도 향상. 확장 단계에서 LLM-as-a-Judge를 추가하여 자동 검증 예정.

### Q2. 비용은 얼마나 드나요?
**A**: MVP 단계 (10일): AWS 인프라 약 $XXX. 확장 단계: LLM API 비용 추가로 월 $XXX 예상.

### Q3. 기존 Redash/Looker와 차이는?
**A**: Redash는 SQL을 알아야 사용 가능. CAPA는 **자연어로 질문**하면 AI가 SQL을 생성하여 SQL 모르는 비개발자도 데이터 조회 가능.

### Q4. 보안은 어떻게?
**A**: Athena **READ-ONLY 권한**으로 제한. SQL 파싱으로 INSERT/DELETE/DROP 차단.

### Q5. MVP 실패 시나리오는?
**A**: 파이프라인 연결만 실패하면 인프라 재검토. Vanna AI 정확도 미달 시 LLM-as-a-Judge 추가 또는 dbt로 Mart 테이블 정제.
