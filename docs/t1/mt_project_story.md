# 프로젝트 스토리 재구성 (개요 확장본)

## 프로젝트 주제
**온라인 광고 로그 파이프라인 구축 및 AI 기반 데이터 분석 헬퍼 설계**

---

## 1. AI 기반 데이터 분석 헬퍼
### 왜 필요한가, 무엇이 달라지는가

### 1-1. 문제 인식: 데이터는 많은데, 분석은 항상 늦다

온라인 광고 시스템에서는 다음과 같은 사용자 행동 데이터가 실시간 로그 형태로 대규모 생성된다.

- 광고 노출 (Impression)
- 클릭 (Click)
- 전환 (Conversion: 가입, 구매 등)

기술적으로는 이미 다음과 같은 환경이 충분히 갖춰져 있다.

- 대규모 실시간 로그 수집 인프라
- 안정적인 로그 저장 및 적재 구조
- SQL 기반 데이터 분석 환경

그러나 실제 업무 현장에서는 다음과 같은 문제가 반복된다.

- “CTR이 왜 떨어졌는지 알고 싶다”
- “어제 캠페인별 성과를 한 번에 보고 싶다”
- “지표 기준을 조금만 바꿔서 다시 보고 싶다”

분석 요청은 자연어로 들어오지만, 이를 실제 분석 결과로 전환하기까지는 다음과 같은 과정이 필요하다.

- 데이터 구조 및 도메인 맥락 파악
- 적절한 테이블 및 컬럼 선택
- SQL 작성 및 실행
- 결과 시각화 및 해석

이 과정에서 가장 큰 병목은 **분석 자체가 아니라 분석 요청과 결과를 주고받는 과정**에 있다.

---

### 1-2. 해결 방향: AI 기반 데이터 분석 헬퍼

본 프로젝트는 기존 SQL 기반 데이터 분석 환경 위에 **AI 기반 데이터 분석 헬퍼**를 결합하는 방향을 제안한다.

핵심 아이디어는 다음과 같다.

> 데이터 분석을 일회성 요청(ticket)이 아니라,  
> 결과를 보며 점진적으로 보완해 나가는 **대화형 과정**으로 전환한다.

---

### 1-3. AI 데이터 분석 헬퍼의 주요 기능

#### (1) 자연어 → 분석 결과까지의 자동 연결

예시:

> “2026-01-23일 동안 캠페인들의 CTR, CVR, RPM을 RPM 역순으로 보여줘.”

- 자연어 요청 해석
- 테이블/컬럼 매핑
- SQL 생성 및 실행
- 차트·그래프 시각화

---

#### (2) 분석 과정의 투명성 확보

- 생성된 SQL 쿼리 제공
- 사용 지표 및 집계 기준 명시
- 결과 검증 가능 구조 유지

---

#### (3) 결과 보완을 위한 대화형 분석

- 광고주별 분리
- 이상치 제거
- 전일 대비 증감 분석

---

## 2. 데이터 파이프라인 최신 기술 적용

### 2-1. 전제 조건: AI 분석은 데이터 품질에 의존한다

AI 분석은 데이터 신뢰성이 전제 조건이다.

---

### 2-2. DBT 도입 배경과 기대 효과

- SQL 변환 로직 버전 관리
- 지표 정의 문서화
- 데이터 테스트 자동화

DBT는 AI 분석 헬퍼의 **신뢰 가능한 기준 레이어** 역할을 한다.

---

### 2-3. 이상 탐지 기법 도입 (Prophet 또는 그에 상응하는 시계열 예측 모델)

온라인 광고 데이터는 변동성이 크다.

#### Prophet 또는 그에 상응하는 시계열 예측 모델 기반 이상 탐지의 역할

- 과거 데이터 기반 기대값 예측  
  (Prophet, ARIMA, LSTM 등 시계열 모델 활용 가능)
- 실제 값과 비교하여 이상 감지
- CTR, CVR, RPM 자동 모니터링

---

## 3. 프로젝트의 핵심 메시지

> AI 분석 헬퍼는 **분석 경험을 개선**하고,  
> 데이터 파이프라인 기술은 **신뢰성을 보장**한다.

본 프로젝트는  
**신뢰 가능한 데이터 위에서 누구나 빠르게 분석할 수 있는 환경**을 목표로 한다.
