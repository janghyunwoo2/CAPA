## 2. 배경 및 문제 정의

### 2.1 현재 상황

- `log-generator/` 광고 로그 시뮬레이터 개발 완료
- `infra/terraform/` 기본 모듈 구조 존재
- 실제 데이터 파이프라인 미구축

### 2.2 해결해야 할 문제

| 문제 | 영향 | 우선순위 |
|------|------|----------|
| 로그 수집 인프라 부재 | 데이터 분석 불가 | 높음 |
| 스키마 미정의 | 쿼리 작성 불가 | 높음 |
| IaC 미구축 | 환경 재현 불가 | 중간 |

## 3. 기술 스택 및 아키텍처 방향

### 3.1 사용 기술

| 영역 | 기술 | 버전 | 선택 이유 |
|------|------|------|-----------|
| 스트리밍 | Kinesis Data Stream | - | AWS 네이티브, Firehose 통합 |
| 적재 | Kinesis Data Firehose | - | Parquet 변환 내장, 동적 파티셔닝 |
| 저장소 | S3 | - | 비용 효율, Athena 통합 |
| 카탈로그 | Glue Data Catalog | - | Athena 통합, 스키마 관리 |
| 쿼리 | Athena | v3 | 서버리스, Parquet 최적화 |
| IaC | Terraform | 1.6+ | 팀 표준, 모듈화 |

### 3.2 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────────────────────┐
```
