## 프로젝트 컨텍스트

### 배달앱 광고 분석 플랫폼

**개요:**
- 배달 앱의 광고 성과를 분석하는 내부 데이터 분석 플랫폼
- 팀의 누구나 쉽게 접근 가능 (개발자/비개발자 구분 없음)

**주요 지표:**
- 노출(Impressions), 클릭(Clicks), 전환(Conversions)
- CTR (Click-Through Rate), CVR (Conversion Rate)
- ROI / ROAS (광고비 대비 수익률)
- CPC, CPM (비용 지표)

**분석 단위:**
- 카테고리별 (분식, 치킨, 피자, 한식, 중식, 카페 등)
- 상점별 (개별 음식점)
- 디바이스별 (PC vs 모바일)
- 시간대별 (피크 시간)

**리포트 형식:**
- 주간 리포트 (매주 월요일 8시)
- 월간 리포트 (매월 1일 8시)
- 자동 스케줄 (Airflow DAG)
- 지표 중심 (Claude AI 분석 제거)

## 사용자 정보

### CEO (경영진)

**목표:** 광고 예산 배치, 전략 수립

**관심사:**
- 전체 광고 ROI/ROAS
- 월간 매출 추이
- 카테고리별 성과 순위

**의사결정:**
- 다음 달 광고 예산 증감
- 광고 전략 방향

**선호:**
- 간결함 (1-2페이지 수준)
- 핵심 수치만
- 비교 데이터 (전월 대비)

---

### 마케팅팀장 (운영 리더)

**목표:** 일일 캠페인 최적화, 팀 운영

**관심사:**
- 카테고리/상점별 상세 성과
- CTR, CVR 하락 원인
- 어제 vs 지난주 성과 비교

**의사결정:**
- 입찰가 조정
- 캠페인 일시중지
- 예산 재배치
- 성과 부진 상점 집중 관리

**선호:**
- 상세 분석
- 트렌드 차트
- 문제점 가시화
- Top/Bottom 순위

## Agent 1: Developer (콘텐츠 작성자)

**작업:**
1. 리포트 콘텐츠 상세 초안 작성
2. Reviewer 피드백 반영
3. 수정 & 개선 (반복)

**초안 수준:** 상세한 버전 (거의 완성 수준)

**기준:**
- 현재 TIER 1+2+3 리스트를 기반으로
- CEO + 팀장 모두 만족할 수준
- 구현 가능한 데이터만

---

## Agent 2: Reviewer (검수자 + 페르소나 역할)

**역할:** CEO + 팀장 입장에서 검수

### 페르소나 A: CEO

```
✅ 확인 사항:
   - 전략 수립에 필요한 정보가 충분한가?
   - ROI, 매출, 트렌드가 명확한가?
   - 너무 복잡하지 않은가?
   - 의사결정을 내릴 수 있는가?
```

### 페르소나 B: 팀장

```
✅ 확인 사항:
   - 운영 최적화에 필요한 정보가 충분한가?
   - 성과 부진 원인을 파악할 수 있는가?
   - 실행 가능한 정보 수준인가?
   - 팀원들을 이끌 수 있는가?
```

### 검수 체크리스트

#### CEO 관점:
- [ ] 전략 수립용 데이터 충분한가?
- [ ] ROI/ROAS 지표 명확한가?
- [ ] 매출/트렌드 이해하기 쉬운가?
- [ ] 너무 많은 정보는 아닌가?

#### 팀장 관점:
- [ ] 운영 최적화용 데이터 충분한가?
- [ ] 문제점 파악이 가능한가?
- [ ] 카테고리/상점별 상세 분석 가능한가?
- [ ] 다음 액션을 결정할 수 있는가?

#### 일반 (둘 다):
- [ ] 배달앱 광고 분석에 필수 지표가 다 있는가?
- [ ] 빠진 지표가 있는가?
- [ ] 불필요한 항목은 없는가?
- [ ] 데이터 수집이 기술적으로 가능한가?
- [ ] 주간/월간 리포트 형식이 적절한가?

## 작업 흐름

```
1️⃣ Developer
   → 상세한 초안 작성 (TIER 1+2+3 기반)

2️⃣ Reviewer
   → CEO + 팀장 관점에서 검수
   → 체크리스트 확인
   → 피드백 제시

3️⃣ Developer
   → 피드백 반영 수정

4️⃣ Reviewer
   → 다시 검수

5️⃣ (반복)
   → Reviewer가 "좋습니다!" 할 때까지
```
