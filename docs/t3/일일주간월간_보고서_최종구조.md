## 1. 개요

**목표**: 순수 지표 중심의 일일/주간/월간 보고서를 통일된 양식으로 제공

**핵심 원칙**:
- 매일 아침 08:00 보고서 발송
- 월별로 누적 데이터 초기화
- 주간은 월요일에만 추가 (월초 부분주차 포함, 월말 불완전주차 포함)
- 월간은 매달 1일에만 표시

#### 주간 섹션 상세 규칙

**월초 부분주차**
- 월 1일이 월요일 아닌 경우, 첫 월요일 전까지를 부분주차로 처리
- 예: 3월 1일(일) → 3/1(일)이 부분주차 → 3/2(월) 아침에 표시

**월말 부분주차**
- 마지막 월요일 이후 남은 날들(월~월말)을 부분주차로 처리
- 예: 3월 30일(월)~31일(화) → 3/30(월) 아침에는 미포함 → 4/1(수) 아침에 추가
- 이유: 주간은 월요일에만 추가되므로, 월말 부분주차는 다음달 1일에 함께 표시

### 3.3 구현 핵심 로직

```python
# main.py
def main(date=None):
    """
    Airflow DAG에서 매일 08:00 호출
    """
    date = date or datetime.now()

    # 1. 날짜 판단: 월요일? 1일?
    is_monday = date.weekday() == 0
    is_month_start = date.day == 1
    month_start = date.replace(day=1)

    # 2. Redash에서 데이터 조회
    if is_month_start:
        # 1일: 전월 데이터 조회 (일간+주간+월간)
        prev_month_end = date - timedelta(days=1)
        prev_month_start = prev_month_end.replace(day=1)
        daily_data = redash_client.get_daily_kpi(prev_month_start, prev_month_end)
        weekly_list = redash_client.get_weekly_list(prev_month_start, prev_month_end)
        monthly_data = redash_client.get_monthly_kpi(...)
    else:
        # 2~31일: 현월 누적 데이터 조회
        daily_data = redash_client.get_daily_kpi(month_start, date)
        weekly_list = redash_client.get_weekly_list(month_start, date) if is_monday else None
        monthly_data = None

    # 3. 마크다운 생성
    markdown = markdown_builder.build(
        date=date,
        daily=daily_data,
        weekly_list=weekly_list,
        monthly=monthly_data
    )

    # 4. PDF 변환 & Slack 발송
    pdf = pdf_exporter.to_pdf(markdown)
    bot.send_to_slack(pdf)

        for week_data in weekly_list:
            md += f"### 주간: {week_data['start_date']}~{week_data['end_date']}\n\n"
            md += format_executive_summary(week_data)
            md += format_kpi_detail(week_data)

    if monthly:

        md += format_executive_summary(monthly)
        md += format_category_perf(monthly)
        md += format_shop_perf(monthly)
        md += format_funnel(monthly)
```
