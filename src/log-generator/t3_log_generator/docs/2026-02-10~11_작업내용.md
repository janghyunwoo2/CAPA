### uv란?

- 가상환경 생성 + 패키지 설치 + 실행을 한 번에

### 설치 과정
```powershell

# 2. PATH 영구 등록
[Environment]::SetEnvironmentVariable("Path", ..., "User")

# 3. VS Code 재시작 필요
```

### 실행 방법
```powershell
cd t3_log_generator

```

## 6️⃣ 마스터 데이터 개념 설명

### 마스터 데이터란?
**"미리 만들어둔 고정된 데이터 풀"**

### 왜 필요한가?

#### 마스터 데이터 없이 (매번 새로 생성)
```python
user_id = str(uuid.uuid4())  # 항상 새로운 사람
```
→ 모든 로그가 다른 사람, 다른 가게 (비현실적)

#### 마스터 데이터 있음 (풀에서 선택)
```python
self.users = [유저 100명 미리 생성]
user_id = random.choice(self.users)
```
→ 같은 유저가 여러 광고를 볼 수 있음 (현실적)

### 예시
```
유저 풀: ["user-A", "user-B", "user-C"]
가게 풀: ["shop-1", "shop-2"]

Impression 1: user_id = "user-A", shop_id = "shop-1"
Impression 2: user_id = "user-A", shop_id = "shop-2"  # 같은 유저!
Impression 3: user_id = "user-B", shop_id = "shop-1"  # 같은 가게!
```

## 8️⃣ 주요 개념 정리

### AWS Kinesis Firehose
- **데이터 전송 특화** 서비스
- 코드 없이 Parquet 변환, 자동 파티셔닝
- S3, Redshift, OpenSearch로 자동 전송

### Lambda vs Firehose
| 항목 | Lambda | Firehose |
|------|--------|----------|
| 목적 | 범용 함수 실행 | 데이터 전송 특화 |
| 코드 | 필수 | 거의 불필요 |
| 변환 | 직접 구현 | 내장 지원 |

### Docker 사용 이유
- AWS 배포 시 필요 (EC2, ECS)
- 로컬 테스트만 할 때는 불필요
- `python main.py`로도 실행 가능


## 3️⃣ AWS 인프라 구축 (Firehose 파이프라인)

### 구성 요소
**Kinesis Stream** (`capa-ad-logs-dev`)
    ↓
**Kinesis Firehose** (`capa-firehose-dev`)
    ↓ (Parquet 변환)
**S3** (`capa-logs-dev-ap-northeast-2`)

### 🔧 주요 설정 (수동)
- **Firehose**: 버퍼 128MB / 300초, 동적 파티셔닝 활성화
- **Glue**: `capa_db.ad_logs` 테이블 생성 및 **스키마 수동 정의**
- **트러블슈팅**:
    - 스키마가 없어서 Parquet 변환 실패 (`DataFormatConversion.NonExistentColumns`)
    - → Glue Console에서 컬럼 직접 추가하여 해결

> 자세한 설정값은 `docs/INFRASTRUCTURE.md`에 기록함.

- 데이터를 나눠 담는 통
- 같은 `user_id`는 같은 샤드로 전송
- 4개 샤드 → 병렬 처리 가능

### PartitionKey
- `user_id`를 사용하여 샤드 결정
- 같은 유저의 로그는 순서 보장

### 비용
