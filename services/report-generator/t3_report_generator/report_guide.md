# CAPA 광고 성과 보고서 보는 법 (가이드)

이 문서는 매일/매주 발송되는 **CAPA 광고 성과 자동 보고서**의 각 지표가 어떤 기준으로 작성되었는지, 명칭들이 실제 보고서의 순서와 똑같이 1:1로 매칭되어 설명되어 있습니다.

---

## 일간 (당월 누적 실적)

보고서를 열자마자 보이는 **"집계 기준"** 블록은 현재 데이터가 **언제부터 언제까지의 누적 실적**인지 안내합니다.

*   **당월 누적 실적**: 매월 1일부터 전날(어제) 자정까지의 누적 성과를 의미합니다.
*   **전기 대비 수치**: 지난달의 동일한 기간(예: 전월 1일~어제 동일 일자)과 비교한 증감률입니다.

> **💡 데이터 원본 파이프라인**
> 보고서의 모든 숫자는 데이터팀이 새벽마다 하루 치 결과를 모아서 만들어두는 **`ad_combined_log_summary` (하루치 요약 테이블)**에서 옵니다.
> 이 요약본은 1시간 단위 보조 테이블들 장부와 마지막 결제(`conversions`) 기록을 모두 모아 하루에 단 한 번만 생성되는 일간 최종본입니다.

---

## Executive Summary (경영진 요약) (일간·주간·월간 공통)

광고를 통해 달성한 **가장 핵심적인 성과**를 한눈에 볼 수 있는 요약판입니다.

### 핵심 KPI 카드
*   **전기 대비**: 각 지표 옆 `(+/-%)` 수치는 전월 동일 기간과 비교한 증감률. 전기 데이터는 `ad_combined_log_summary`에서 날짜 범위만 바꿔 별도 조회
*   **광고 기여 매출**: 오직 **"우리 광고 배너를 클릭하고 들어온 고객이 결제했을 때의 영수증 금액"**만을 모두 합친 순수 기여 매출입니다. (앱 전체의 자연 발생 매출 제외)
    *   **데이터 출처**: `ad_combined_log_summary` 테이블의 `conversion_value` 컬럼
    *   **계산식**: `SUM(conversion_value)` — `conversion_type = 'purchase'` (실제 결제) AND `is_click = TRUE` (광고 클릭)인 건만 포함
    *   **conversion_type** (전환 유형): 고객이 광고 클릭 후 한 행동의 종류. `purchase`(결제), `add_to_cart`(장바구니 담기), `view_content`(메뉴 조회), `signup`(회원가입), `download`(앱 다운로드) 등이 있으나 돈이 오가는 건 `purchase`뿐
    *   **conversion_value** (전환 금액): 고객이 최종 결제한 주문 합계금액. `conversions` 테이블에서 `impression_id` 기준으로 조인되어 들어온 값
*   **총 ROAS** (광고 수익률): `(광고 기여 매출 ÷ 광고비) × 100`. (예: 1원의 광고비로 3원의 매출을 냈다면 300%)
    *   **계산식**: `SUM(purchase의 conversion_value) / SUM(cost_per_impression + cost_per_click) × 100`
    *   **광고비 구성**: `cost_per_impression` (노출 비용) + `cost_per_click` (클릭 비용)
    *   **⚠️ 과금 방식 미확정**: 배달앱 업계 표준은 클릭비(CPC)만 청구하는 경우가 많음 (배민, 쿠팡이츠 등). 현재는 노출비+클릭비 둘 다 합산하는 방식으로 운영 중이나, 추후 CPC 방식으로만 변경 가능성 있음. 확정 시 `cost_per_impression` 제거 필요.
*   **총 CTR** (클릭률): `(클릭 수 ÷ 노출 수) × 100`. 광고 소재(이미지)가 고객에게 얼마나 매력적인지 나타냅니다.
    *   **계산식**: `SUM(is_click=TRUE) / COUNT(DISTINCT impression_id) × 100`
    *   **컬럼**: `is_click` (클릭 여부) ÷ `impression_id` (노출 고유 ID)
*   **총 전환**: 광고를 누르고 들어온 고객이 최종적으로 구매 버튼까지 누른 '결제 건수'입니다.
    *   **계산식**: `COUNT(conversion_type = 'purchase' AND is_click = TRUE)`
    *   **컬럼**: `conversion_type` (전환 유형) + `is_click` (광고 클릭 여부) — 둘 다 만족하는 건만 카운트

### 보조 KPI
*   **전기 대비**: 각 수치 옆 `(+/-%)` 는 전월 동일 기간과 비교한 증감률. 핵심 KPI 카드와 동일한 전기 데이터 사용
*   **노출**: 광고 영상이나 이미지가 고객 화면에 뜬 총 횟수. (볼륨)
    *   **계산식**: `COUNT(DISTINCT impression_id)`
    *   **컬럼**: `impression_id` (노출 고유 ID) — 중복 제거 후 카운트
*   **클릭**: 고객이 광고에 흥미를 느껴 누른 총 횟수.
    *   **계산식**: `SUM(is_click = TRUE)`
    *   **컬럼**: `is_click` (클릭 여부, TRUE/FALSE) — TRUE인 건만 합산
*   **광고비**: 노출 및 클릭으로 실제 지출된 마케팅 비용 총계.
    *   **계산식**: `SUM(cost_per_impression + cost_per_click)`
    *   **컬럼**: `cost_per_impression` (노출 비용) + `cost_per_click` (클릭 비용, 클릭 없으면 NULL)
*   **순이익**: `(광고 기여 매출) - (광고비)`. 광고 한 번으로 실질적으로 떨어진 마진입니다.
    *   **계산식**: `revenue - cost`
    *   **revenue** (광고 기여 매출): `SUM(conversion_value)` — `conversion_type = 'purchase'` AND `is_click = TRUE`인 건만
    *   **cost** (광고비): `SUM(cost_per_impression + cost_per_click)` — 노출 비용 + 클릭 비용 합산

## KPI 상세 (일간·주간·월간 공통)

이번 기간과 전기(지난달 동일 기간)를 나란히 놓고 비교하는 종합 성적표입니다.

*   **전기 기간 계산 방식**: 이번 기간 시작일에서 하루 뺀 날의 월초~말일을 전기로 사용
    *   예) 이번 기간 3/1~3/10 → 전기 = 2/1~2/10 (전월 동일 기간)
    *   전기 데이터는 동일 테이블(`ad_combined_log_summary`)에서 `WHERE 날짜 BETWEEN 전기시작 AND 전기끝` 으로 날짜 범위만 바꿔 별도 조회 후 비교
    *   **전기 데이터를 사용하는 섹션**: Executive Summary (핵심 KPI 카드 전기 대비), KPI 상세 (모든 지표 전기 비교)

| 지표 | 계산식 | 사용 컬럼 |
|-----|------|---------|
| **노출** | `COUNT(DISTINCT impression_id)` | `impression_id` |
| **클릭** | `SUM(is_click = TRUE)` | `is_click` |
| **전환** | `COUNT(conversion_type = 'purchase' AND is_click = TRUE)` | `conversion_type`, `is_click` |
| **광고 기여 매출** | `SUM(conversion_value)` — purchase + 클릭 건만 | `conversion_value`, `conversion_type`, `is_click` |
| **광고비** | `SUM(cost_per_impression + cost_per_click)` | `cost_per_impression`, `cost_per_click` |
| **순이익** | `광고 기여 매출 - 광고비` | (위 두 값으로 계산) |
| **CTR** (클릭률) | `클릭 수 / 노출 수 × 100` | `is_click`, `impression_id` |
| **CVR** (구매 전환율) | `결제 건수 / 클릭 수 × 100` | `conversion_type`, `is_click` |
| **CPC** (클릭당 비용) | `SUM(cost_per_click) / 클릭 수` | `cost_per_click`, `is_click` |
| **ROAS** (광고 수익률) | `광고 기여 매출 / 광고비 × 100` | (위 두 값으로 계산) |

## 일별 트렌드 (일간 섹션 전용)

### 일별 상세 테이블
이번 달 1일부터 어제까지의 **매일매일의 성과 변화**를 표 형태로 보여줍니다.
*   "어느 요일에 광고를 통한 결제 누적액이 튀었는가?"
*   "특정 이벤트 날짜의 일간 ROAS는 어땠는가?" 등을 추적할 수 있습니다.

| 컬럼 | 설명 | 출처 |
|-----|------|------|
| **날짜** | 해당 일자 | `date` (impression_timestamp에서 변환) |
| **요일** | 날짜 기준 요일 | 날짜로부터 계산 (별도 컬럼 없음) |
| **노출** | 해당 일 노출 수 | `COUNT(DISTINCT impression_id)` |
| **클릭** | 해당 일 클릭 수 | `SUM(is_click = TRUE)` |
| **전환** | 해당 일 결제 건수 | `COUNT(purchase AND is_click = TRUE)` |
| **광고 기여 매출** | 해당 일 결제 금액 합계 | `SUM(conversion_value)` — purchase + 클릭 건만 |
| **광고비** | 해당 일 노출+클릭 비용 | `SUM(cost_per_impression + cost_per_click)` |
| **CTR** | 해당 일 클릭률 | `클릭 수 / 노출 수 × 100` |
| **ROAS** | 해당 일 광고 수익률 | `광고 기여 매출 / 광고비 × 100` |

## 주간 섹션 (월요일에만 추가)

*   **발송 조건**: 월요일에만 일간 보고서에 추가됨
*   **범위**: 월초부터 지난 주까지 완성된 주차들 (월~일 기준)
    *   월초 부분주차 포함 (예: 3/1이 일요일이면 3/1~3/1도 1주차로 포함)
    *   월말 불완전주차는 다음달 1일에 추가됨
*   **전기**: 전주 데이터와 비교 (일간의 "전월"과 다름)
*   **포함 섹션**: Executive Summary (전주 대비), KPI 상세 (전주 비교) — 일별 트렌드는 없음

## 상점별 성과 (월간 섹션 전용)

> **⚠️ 미검증**: 월간 보고서를 실제로 생성할 수 있을 때 (한 달치 데이터 쌓이면) 직접 보고서 확인 후 가이드 내용 수정 필요

개별 **가게 단위**의 특화된 광고 효율 표입니다.

### Top 10 상점 (매출 기준)
*   우리 측 광고 시스템을 가장 잘 활용하여 돈을 쓸어 담고 있는 '우수 고객' 10곳입니다.
*   **컬럼**: 순위, 상점 ID (`store_id`), 카테고리 (`food_category`), 노출, 클릭, 전환, 광고 기여 매출, CTR, ROAS, 매출 변화
*   **정렬 기준**: 매출 내림차순
*   **매출 변화**: 현재 `+0.0%` 하드코딩 — 데이터 쌓이면 전월 대비로 수정 필요

### Bottom 10 상점 (ROAS 기준, 최소 노출 100건)
*   **노출은 100번 이상** 충분히 일어났으나, 구매 전환이 너무 떨어져 **ROAS(수익률)가 처참한 10곳**입니다.
*   **컬럼**: 순위, 상점 ID, 카테고리, 노출, 클릭, 전환, 광고 기여 매출, CTR, ROAS, 상태
*   **상태 기준**: ROAS < 100% → `손실`, ROAS < 150% → `요주의`, 그 외 → `개선 중`
*   **정렬 기준**: ROAS 오름차순, 최소 노출 100건 이상
