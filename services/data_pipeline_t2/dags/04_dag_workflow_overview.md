# CAPA T2 Data Pipeline - 01-03 DAG Workflow 설명

01번부터 03번까지의 DAG는 **데이터 품질 관리, 성능 최적화, 메타데이터 거버넌스**를 담당하는 전문화된 파이프라인입니다.

## 🔄 01-03 DAG 워크플로우

```
01_data_quality_dag.py (데이터 품질 검사)
    ↓  
02_preagg_dag.py (성능 최적화 집계)
    ↓
03_schema_extraction_dag.py (메타데이터 관리)
```

---

## 📊 **01_data_quality_dag.py - 데이터 품질 관리**

### **핵심 역할**
- 원시 데이터의 **품질 검증** 및 **이상 탐지**
- 데이터 파이프라인의 **신뢰성 보장**

### **품질 체크 항목**
- **NULL 비율 검사**: 필수 필드의 결측값 비율 모니터링
- **데이터 타입 검증**: 예상 스키마와 실제 데이터 타입 일치 여부
- **중복 데이터 체크**: 동일 이벤트의 중복 발생 탐지
- **통계적 이상치**: 평소 패턴과 크게 다른 데이터 감지

### **출력 결과**
- S3에 일일 품질 보고서 JSON 저장
- 품질 지표 대시보드용 메트릭 생성

---

## ⚡ **02_preagg_dag.py - 대용량 집계 최적화**

### **핵심 역할**  
- **AWS Athena** 기반 대용량 로그 데이터 사전 집계
- **쿼리 성능 향상** 및 **비용 절감**

### **집계 처리**
- **일일 집계**: 광고주별 일일 성과 KPI (impressions, clicks, conversions)
- **시간별 집계**: 세분화된 시계열 분석용 데이터
- **Parquet + 파티셔닝**: 날짜별 파티션으로 스캔 범위 최소화

### **기술적 최적화**
- **CTAS 쿼리**: 원본 데이터 스캔 → 집계 테이블 생성
- **압축률**: 일반적으로 200:1 압축으로 저장 공간 절약
- **쿼리 비용**: 전체 로그 스캔 대신 집계 테이블 조회로 비용 대폭 절감

---

## 🗃️ **03_schema_extraction_dag.py - 메타데이터 카탈로그**

### **핵심 역할**
- **데이터베이스 스키마** 자동 추출 및 버전 관리
- **데이터 거버넌스** 및 **변경 이력 추적**

### **추출 정보**
- **테이블 구조**: 모든 테이블의 컬럼명, 데이터타입, 제약조건
- **메타데이터**: Primary Key, NULL 허용 여부 등 상세 정보
- **일일 스냅샷**: 날짜별 스키마 변경사항 추적

### **활용 시나리오**
- **영향도 분석**: 스키마 변경이 다운스트림에 미치는 영향 파악
- **문서 자동화**: 데이터 사전 자동 생성 및 업데이트
- **호환성 체크**: API/ETL 파이프라인과의 스키마 호환성 검증

---

## 🔗 **01-03 DAG 간 상호 작용**

### **데이터 흐름**
1. **01번**: 원시 데이터 품질 검증 → 품질 통과 시 다음 단계 진행
2. **02번**: 검증된 데이터를 대상으로 효율적인 집계 테이블 생성  
3. **03번**: 처리 과정에서 사용된 스키마 정보를 메타데이터로 보존

### **운영 시너지**
- **품질 기반 처리**: 01번의 품질 체크 통과 시에만 02번 집계 실행
- **메타데이터 연동**: 03번의 스키마 정보로 01-02번 DAG의 검증 로직 개선
- **장애 격리**: 각 DAG 독립 실행으로 부분 실패 시에도 전체 파이프라인 중단 방지

### **비즈니스 가치**
- **데이터 신뢰성**: 01번으로 높은 품질의 데이터 보장
- **운영 효율성**: 02번으로 분석 쿼리 성능 및 비용 최적화  
- **거버넌스**: 03번으로 데이터 자산 체계적 관리

이 3개 DAG는 **현대적인 데이터 파이프라인의 핵심 요구사항**인 **품질(Quality), 성능(Performance), 거버넌스(Governance)**를 각각 전담하여 안정적이고 확장 가능한 데이터 시스템을 구축합니다.
