# 03_schema_extraction_dag.py 설명서

`03_schema_extraction_dag.py` 파일은 데이터베이스에서 스키마 정보를 자동으로 추출하여 S3에 저장하는 메타데이터 관리 시스템을 위한 Airflow DAG를 정의합니다.

## 1. 개요 (Overview)
- **DAG ID**: `03_schema_extraction`
- **주요 목적**: PostgreSQL 데이터베이스의 모든 테이블과 컬럼 메타데이터를 추출하여 JSON 형태로 S3에 저장함으로써 데이터 카탈로그 및 스키마 변경 추적을 가능하게 합니다.
- **실행 주기**: `@daily` (매일 1회 실행)

## 2. 주요 태스크 구성 (Workflow)

스키마 추출 프로세스는 다음과 같은 단계로 진행됩니다:

1.  **`extract_schema`**: 
    - 이 DAG의 핵심 태스크로, SQLAlchemy를 사용하여 데이터베이스에 연결하고 스키마 정보를 추출합니다.
    - **추출 정보**: 각 테이블별로 다음 메타데이터를 수집합니다.
        - 컬럼명 (`name`)
        - 데이터 타입 (`type`)
        - NULL 허용 여부 (`nullable`)
        - 기본키 여부 (`primary_key`)
    - **저장 형식**: 추출된 스키마 정보를 JSON 형태로 구조화하여 S3에 업로드합니다.
    - **파일명**: `metadata/schema_{YYYYMMDD}.json` 형식으로 날짜별 관리

## 3. 기술적 특징

### 패키지 의존성 안전 처리
코드 내에서 `HAS_SQLALCHEMY` 및 `HAS_BOTO3` 플래그를 통해 필수 패키지의 설치 여부를 체크하고, 패키지가 없는 환경에서도 샘플 데이터로 대체하여 실행할 수 있도록 설계되었습니다.

### 다중 계층 오류 처리
1. **SQLAlchemy 미설치**: 샘플 스키마 데이터 반환
2. **데이터베이스 연결 실패**: 광고 관련 샘플 스키마로 대체
3. **S3 업로드 실패**: 로컬에서 스키마 추출만 완료 처리

### 연결 타임아웃 설정
데이터베이스 연결 시 `connect_timeout=5` 설정으로 무한 대기를 방지하여 안정성을 확보합니다.

## 4. 관련 환경 변수
- **데이터베이스 설정**:
  - `DB_USER`: 데이터베이스 사용자명 (기본값: `testuser`)
  - `DB_PASS`: 데이터베이스 비밀번호 (기본값: `testpass`)
  - `DB_HOST`: 데이터베이스 호스트 (기본값: `localhost`)
  - `DB_PORT`: 데이터베이스 포트 (기본값: `5432`)
  - `DB_NAME`: 데이터베이스 이름 (기본값: `testdb`)
- **S3 설정**:
  - `S3_BUCKET`: 스키마 메타데이터가 저장될 S3 버킷 이름 (기본값: `capa-logs-dev-ap-northeast-2`)

## 5. 출력 데이터 구조

추출된 스키마는 다음과 같은 JSON 구조로 저장됩니다:

```json
{
  "table_name_1": [
    {
      "name": "id",
      "type": "BIGINT",
      "nullable": false,
      "primary_key": true
    },
    {
      "name": "advertiser_id", 
      "type": "INTEGER",
      "nullable": true,
      "primary_key": false
    }
  ],
  "table_name_2": [
    // 컬럼 정보...
  ]
}
```

## 6. 비즈니스 가치
- **데이터 거버넌스**: 데이터베이스 스키마 변경 이력을 자동으로 추적하여 데이터 품질 관리에 기여합니다.
- **문서화 자동화**: 수동으로 관리하던 데이터 사전을 자동화하여 개발팀의 생산성을 향상시킵니다.
- **호환성 체크**: 스키마 변경이 다운스트림 시스템에 미치는 영향을 사전에 파악할 수 있습니다.
- **메타데이터 카탈로그**: 다른 분석 도구나 데이터 파이프라인에서 참조할 수 있는 중앙화된 스키마 정보를 제공합니다.
